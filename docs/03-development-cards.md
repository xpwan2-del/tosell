# 功能卡片与开发计划

## 1. 里程碑总览

| 阶段 | 名称 | 建议周期 | 目标 |
| --- | --- | --- | --- |
| 阶段 0 | 需求确认与规则定稿 | 3-5 天 | 定清业务、账务、状态机、权限和原型 |
| 阶段 1 | 用户端小程序交易闭环 | 2 周 | 用户能进入代理店铺并完成购买 |
| 阶段 2 | 代理端经营闭环 | 2 周 | 代理能入驻、配置店铺、上架商品、看收益 |
| 阶段 3 | 平台后台管理闭环 | 2-3 周 | 平台能审核、管理、履约、售后、结算 |
| 阶段 4 | 财务结算与风控闭环 | 1-2 周 | 退款、冻结、结算、追扣、对账可用 |
| 阶段 5 | 测试、提审和灰度上线 | 1-2 周 | 小范围真实交易跑通后上线 |
| 阶段 6 | H5 独立店铺与多端统一 | 2 周 | H5 能在微信外浏览器打开并复用同一后端 |
| 阶段 7 | 受控两级渠道供货 | 2-3 周 | 一级给二级配置转供价，二级独立销售 |
| 阶段 8 | 多渠道支付与生产部署规划 | 1-2 周 | 预留微信小程序、微信 H5、支付宝和服务器部署方案 |

## 2. 阶段 0：需求确认与规则定稿

### 卡 0.1：业务规则确认

目标：确认平台供货、代理小店、自由定价、保证金、服务费、退款和结算规则。

交付物：

1. 商品规则说明。
2. 定价规则说明。
3. 服务费规则说明。
4. 退款规则说明。
5. 结算规则说明。

验收标准：

1. 能用具体订单例子算出平台收入、代理收益和服务费。
2. 能解释全额退款、部分退款、已结算后退款如何处理。
3. 能明确哪些订单能结算，哪些订单必须冻结。

### 卡 0.2：角色权限表

目标：定义用户、代理、运营、财务、管理员的权限边界。

交付物：

1. 角色列表。
2. 菜单权限。
3. 数据权限。
4. 操作权限。

验收标准：

1. 代理不能看到其他代理数据。
2. 财务权限和运营权限分离。
3. 敏感操作需要管理员或指定角色处理。

### 卡 0.3：状态机设计

目标：设计订单、退款、履约、结算和店铺状态。

交付物：

1. 订单状态机。
2. 退款状态机。
3. 履约状态机。
4. 结算状态机。
5. 店铺状态机。

验收标准：

1. 每个状态能否退款明确。
2. 每个状态能否结算明确。
3. 每个状态能否履约明确。
4. 异常状态有处理路径。

### 卡 0.4：页面原型

目标：输出低保真页面结构，避免开发阶段反复改方向。

交付物：

1. 用户端页面结构。
2. 代理端页面结构。
3. 平台后台页面结构。

验收标准：

1. 每个核心流程页面完整。
2. 页面字段和业务规则对应。
3. 开发可以基于页面结构拆任务。

### 卡 0.5：H5 与两级渠道口径定稿

目标：确认小程序 + H5 + 受控两级渠道供货的产品边界。

交付物：

1. H5 买家端页面范围。
2. 一级渠道商和二级渠道商角色说明。
3. 渠道授权规则。
4. 平台供货价、一级转供价、最终销售价计算口径。
5. 禁止三级渠道、返佣、团队奖、邀请奖励和代理等级的边界说明。

验收标准：

1. 能用 `平台 100 -> 一级 110 -> 二级 150` 的例子复算一级差价和二级差价。
2. 能明确二级渠道商不能继续创建三级渠道。
3. 能明确哪些页面、接口、字段不得出现返佣或团队收益口径。

## 3. 阶段 1：用户端小程序交易闭环

### 卡 1.1：代理店铺首页

功能：

1. 展示代理店铺名称、头像、简介、公告。
2. 展示代理客服微信。
3. 展示本店已上架商品。
4. 支持分享店铺链接或二维码。

验收标准：

1. 用户通过代理链接进入后看到代理自己的小店。
2. 店铺商品只展示该代理已上架商品。
3. 分享路径能携带店铺标识。

### 卡 1.2：商品详情页

功能：

1. 展示商品名称、售价、图片、权益说明。
2. 展示有效期、使用规则、发放方式和售后规则。
3. 支持立即购买。

验收标准：

1. 售价为代理设置的销售价。
2. 不展示平台供货价。
3. 下架商品不能购买。

### 卡 1.3：下单支付

功能：

1. 创建订单。
2. 后端计算订单金额。
3. 拉起微信支付或虚拟支付。
4. 处理支付成功、取消和失败。

验收标准：

1. 订单绑定正确代理。
2. 订单金额以后端计算为准。
3. 支付回调成功后订单状态正确变化。
4. 重复回调不会重复处理。

### 卡 1.4：订单中心

功能：

1. 展示用户订单列表。
2. 展示订单详情。
3. 展示履约状态和权益信息。
4. 支持发起退款/售后。

验收标准：

1. 用户只能看到自己的订单。
2. 订单状态与后台一致。
3. 履约成功后用户能看到权益结果。

### 卡 1.5：售后申请

功能：

1. 用户选择售后原因。
2. 用户填写说明。
3. 用户提交凭证。
4. 用户查看售后进度。

验收标准：

1. 退款中订单自动冻结结算。
2. 已退款订单不能重复申请。
3. 售后记录可被平台后台查看。

## 4. 阶段 2：代理端经营闭环

### 卡 2.1：代理入驻申请

功能：

1. 填写身份或主体信息。
2. 填写联系方式。
3. 填写客服微信。
4. 填写售后团队说明。
5. 查看审核状态。

验收标准：

1. 未审核代理不能开店。
2. 审核拒绝需要记录原因。
3. 审核通过后进入保证金环节。

### 卡 2.2：保证金管理

功能：

1. 展示应缴保证金。
2. 展示缴纳状态。
3. 展示扣减记录。
4. 展示退还状态。

验收标准：

1. 未缴纳保证金不能销售。
2. 保证金扣减必须有原因和记录。
3. 保证金不足时可触发限制。

### 卡 2.3：店铺配置

功能：

1. 设置店铺名称。
2. 上传店铺头像或 Logo。
3. 设置店铺公告。
4. 绑定客服微信。
5. 设置售后说明。
6. 获取店铺分享二维码。

验收标准：

1. 店铺资料保存后用户端可见。
2. 客服微信可复制或展示。
3. 被禁用店铺不能继续销售。

### 卡 2.4：平台商品选品

功能：

1. 查看平台商品库。
2. 查看供货价、最低限价、建议售价。
3. 选择商品上架。
4. 下架已选商品。

验收标准：

1. 平台下架商品不可被代理继续上架。
2. 已上架商品出现在代理店铺。
3. 下架后用户端不可购买。

### 卡 2.5：代理自主定价

功能：

1. 代理设置销售价。
2. 系统校验最低限价。
3. 展示预估收益。

验收标准：

1. 低于最低限价保存失败。
2. 等于或高于最低限价保存成功。
3. 预估收益公式正确。

### 卡 2.6：代理订单和收益

功能：

1. 查看本店订单。
2. 查看订单金额、供货价、服务费、预计收益。
3. 查看待结算、冻结中、可结算、已结算金额。
4. 查看退款扣减和追扣记录。

验收标准：

1. 代理只能看到自己的订单。
2. 收益计算与后台一致。
3. 退款中订单不进入可结算。

### 卡 2.7：代理自有商品提交与审核

功能：

1. 代理填写自有商品资料。
2. 上传商品图片和详情。
3. 设置价格和售后规则。
4. 提交平台审核。
5. 查看审核状态。

验收标准：

1. 未审核通过不能售卖。
2. 审核拒绝需要记录原因。
3. 平台可以下架代理自有商品。
4. 平台可以基于风控策略限制部分代理提交或销售自有商品。

## 5. 阶段 3：平台后台管理闭环

### 卡 3.1：代理审核

功能：

1. 查看代理申请列表。
2. 审核通过或拒绝。
3. 查看保证金状态。
4. 冻结、解冻、禁用店铺。

验收标准：

1. 审核动作有操作日志。
2. 冻结店铺不能新增销售。
3. 禁用店铺入口需要有合理提示。

### 卡 3.2：平台商品库

功能：

1. 创建商品。
2. 设置供货价、最低限价、建议售价。
3. 设置履约方式和售后规则。
4. 上架和下架商品。

验收标准：

1. 金额字段精度正确。
2. 下架商品不能被购买。
3. 价格变化不影响历史订单快照。

### 卡 3.3：代理商品审核

功能：

1. 查看代理提交商品。
2. 审核通过或拒绝。
3. 下架违规商品。
4. 标记风险原因。

验收标准：

1. 未审核商品不展示。
2. 审核记录可追溯。
3. 下架后不可继续购买。

### 卡 3.4：订单管理

功能：

1. 查看全平台订单。
2. 按代理、商品、状态筛选。
3. 查看支付、履约、退款和结算状态。
4. 手动冻结订单。
5. 添加订单备注。

验收标准：

1. 平台能追踪每笔订单来源。
2. 冻结订单不能结算。
3. 订单备注和操作记录可追溯。

### 卡 3.5：履约管理

功能：

1. 查看待履约订单。
2. 自动或人工发放虚拟权益。
3. 记录发放凭证。
4. 失败重试。
5. 人工补发。

验收标准：

1. 发放成功后订单状态正确。
2. 发放失败可重试或补发。
3. 系统避免重复发放。

### 卡 3.6：退款售后管理

功能：

1. 查看售后申请。
2. 审核退款。
3. 标记责任归属。
4. 设置退款金额。
5. 处理代理差价扣减和平台供货价扣减。

验收标准：

1. 退款金额不能超过用户实付金额。
2. 退款中订单冻结结算。
3. 责任归属必须记录。

### 卡 3.7：结算管理

功能：

1. 查看待结算订单。
2. 查看冻结订单。
3. 生成结算单。
4. 导出人工打款文件。
5. 回填结算状态。

验收标准：

1. 同一订单不能重复结算。
2. 结算单金额等于订单明细汇总。
3. 已结算后退款可生成追扣记录。

### 卡 3.8：数据看板

功能：

1. GMV。
2. 订单数。
3. 平台供货价金额。
4. 平台服务费。
5. 代理差价收益。
6. 退款率。
7. 履约成功率。
8. 活跃代理数。

验收标准：

1. 指标口径清晰。
2. 能按日期筛选。
3. 能按代理或商品查看基础明细。

## 6. 阶段 4：财务结算与风控闭环

### 卡 4.1：订单拆账

功能：

1. 记录用户实付金额。
2. 记录平台供货价。
3. 记录平台服务费。
4. 记录代理预计收益。
5. 记录金额快照。

验收标准：

1. 历史订单不受后续价格调整影响。
2. 金额计算和展示一致。
3. 财务可导出明细。

### 卡 4.2：退款拆账

功能：

1. 按用户实付金额退款。
2. 拆分平台承担部分。
3. 拆分代理承担部分。
4. 处理服务费退还。
5. 生成退款账务记录。
6. 在售后单记录责任归属：平台责任、代理责任、用户原因或混合责任。
7. 在售后单记录平台承担金额、代理承担金额和服务费承担方。

验收标准：

1. 全额退款金额正确。
2. 部分退款金额正确。
3. 退款后收益和结算状态正确。
4. 退款责任和承担金额可追溯。
5. 已结算订单退款可进入追扣流程。

### 卡 4.3：追扣机制

功能：

1. 已结算后退款生成追扣。
2. 从未结算收益扣减。
3. 从待打款收益扣减。
4. 从保证金扣减。
5. 保证金不足时限制店铺。

验收标准：

1. 追扣记录可追溯。
2. 不允许金额被扣成不明状态。
3. 代理端能看到扣减原因。

### 卡 4.4：风控冻结

功能：

1. 人工冻结订单。
2. 人工解冻订单。
3. 冻结店铺。
4. 禁用店铺。
5. 限制结算。

验收标准：

1. 冻结原因必须填写。
2. 冻结期间不能结算。
3. 解冻后按规则重新进入结算流程。

## 7. 阶段 5：测试、提审和灰度上线

### 卡 5.1：核心链路联调

范围：

1. 代理入驻。
2. 保证金确认。
3. 店铺配置。
4. 商品上架。
5. 用户购买。
6. 支付成功。
7. 权益发放。
8. 收益生成。

验收标准：

1. 完整链路能跑通。
2. 数据状态一致。
3. 金额计算正确。

### 卡 5.2：退款结算联调

范围：

1. 未履约退款。
2. 已履约退款。
3. 退款中冻结。
4. T+1 可结算。
5. 已结算后追扣。

验收标准：

1. 退款金额正确。
2. 结算金额正确。
3. 冻结逻辑正确。

### 卡 5.3：权限和数据隔离测试

范围：

1. 用户订单隔离。
2. 代理订单隔离。
3. 平台角色权限。
4. 敏感操作日志。

验收标准：

1. 无越权访问。
2. 无跨代理数据泄露。
3. 敏感操作可追溯。

### 卡 5.4：小程序提审准备

范围：

1. 小程序类目。
2. 支付能力。
3. 隐私协议。
4. 用户协议。
5. 售后规则。
6. 商品说明。

验收标准：

1. 提审资料完整。
2. 商品内容与类目匹配。
3. 支付方式符合虚拟商品规则。

### 卡 5.5：灰度试运营

范围：

1. 3-5 个测试代理。
2. 5-10 个平台商品。
3. 小额真实支付。
4. 真实退款。
5. 真实结算。
6. 客服话术演练。

验收标准：

1. 真实订单闭环可用。
2. 财务对账无明显差异。
3. 售后处理流程顺畅。
4. 未出现重大投诉或履约问题。

## 8. 阶段 6：H5 独立店铺与多端统一

### 卡 6.1：H5 店铺首页

功能：

1. 浏览器打开独立 H5 店铺链接。
2. 展示店铺名称、公告、客服微信或联系方式。
3. 展示本店已上架商品。
4. 支持 H5 分享链接和二维码。

验收标准：

1. H5 链接可在微信外浏览器打开。
2. H5 店铺和小程序店铺读取同一店铺数据。
3. 用户端不展示供货价、转供价、服务费、渠道差价和结算金额。

### 卡 6.2：H5 商品详情与下单

功能：

1. 展示商品名称、售价、权益说明、售后规则。
2. 创建订单。
3. 进入支付占位页。
4. 查看订单详情。

验收标准：

1. H5 下单与小程序下单调用同一后端订单能力。
2. 订单金额以后端计算为准。
3. 下架商品、风控店铺、低于最低限价商品不能购买。

### 卡 6.3：H5 用户身份

功能：

1. 支持 H5 用户登录占位。
2. 预留手机号/短信验证码或网页登录身份。
3. 用户订单与权益按 H5 身份隔离。
4. 小程序 `openid/unionid` 与 H5 身份可后续合并。

验收标准：

1. H5 用户只能访问自己的订单和权益。
2. 小程序用户身份和 H5 用户身份不会互相串单。
3. 登录态过期后敏感接口拒绝访问。

### 卡 6.4：H5 售后与权益

功能：

1. 查看权益发放结果。
2. 提交售后申请。
3. 上传凭证。
4. 查看售后进度。

验收标准：

1. H5 售后进入同一售后后台。
2. 退款中订单冻结结算。
3. 用户只能查看自己的售后记录。

## 9. 阶段 7：受控两级渠道供货

### 卡 7.1：一级渠道能力申请

功能：

1. 一级渠道商提交开通二级渠道能力申请。
2. 平台查看申请资料、保证金、风险状态和经营记录。
3. 平台审核通过或拒绝。
4. 平台可关闭一级的二级渠道能力。

验收标准：

1. 未授权一级不能创建二级渠道商。
2. 审核动作有审计日志。
3. 关闭能力后不可新增二级渠道关系，历史订单不受影响。

### 卡 7.2：二级渠道商入驻

功能：

1. 二级渠道商提交入驻申请。
2. 绑定所属一级渠道商。
3. 平台审核二级资质。
4. 二级缴纳保证金并开通店铺。

验收标准：

1. 二级必须审核通过并缴纳保证金后才能销售。
2. 二级拥有独立店铺、客服、订单、售后和结算。
3. 二级不能继续创建三级渠道商。

### 卡 7.3：转供商品与转供价

功能：

1. 一级选择可转供给二级的商品。
2. 一级设置一级转供价。
3. 后端校验一级转供价不得低于平台供货价。
4. 二级设置最终销售价。
5. 后端校验最终销售价不得低于一级转供价和平台限制价。

验收标准：

1. 低于平台供货价的一级转供价保存失败。
2. 低于一级转供价的二级最终售价保存失败。
3. 转供价调整写审计日志，且不影响历史订单。

### 卡 7.4：供货链订单快照

功能：

1. 订单创建时固化平台、一级、二级、店铺和商品信息。
2. 固化平台供货价、一级转供价、最终售价、服务费、一级差价、二级差价。
3. 固化渠道授权状态和价格规则版本。
4. 支付成功后进入统一履约流程。

验收标准：

1. 示例订单可复算：平台 100，一级转供 110，二级售价 150，一级差价 10，二级差价 40。
2. 后续调价不影响历史订单快照。
3. 订单归属、收益和结算均以后端快照为准。

### 卡 7.5：一级/二级结算

功能：

1. 一级渠道差价进入一级待结算。
2. 二级销售差价进入二级待结算。
3. T+1 从履约成功时间起算。
4. 退款中、投诉中、风控冻结、履约失败订单不得结算。
5. 分别生成一级和二级结算明细。

验收标准：

1. 同一订单不会重复结算。
2. 一级和二级结算金额可独立导出。
3. 人工打款回填和审计日志完整。

### 卡 7.6：两级退款与追扣

功能：

1. 售后单记录责任归属：平台、一级、二级、用户、混合责任。
2. 记录平台承担、一级承担、二级承担和服务费承担方。
3. 未结算订单直接冻结和扣减待结算收益。
4. 已结算订单生成一级/二级追扣。
5. 追扣不足时按保证金规则限制店铺或结算。

验收标准：

1. 退款金额不超过用户实付金额。
2. 一级责任和二级责任可以分别追扣。
3. 追扣顺序为未结算收益 -> 待打款收益 -> 保证金。

## 10. 阶段 8：多渠道支付与生产部署规划

### 卡 8.1：支付渠道抽象

功能：

1. 设计统一 `payment_channel`。
2. 预留微信小程序支付。
3. 预留微信 H5/JSAPI 支付。
4. 预留支付宝手机网站支付。
5. mock 支付保留本地闭环。

验收标准：

1. 支付单、回调、退款单、退款回调使用统一模型。
2. mock 支付不得跳过订单、ledger、退款和结算逻辑。
3. 真实渠道接入前页面清楚标记“待开通/模拟支付”。

### 卡 8.2：生产服务器与域名规划

功能：

1. 规划 API 域名。
2. 规划 H5 店铺域名。
3. 规划后台域名。
4. 配置 HTTPS、备案、小程序合法域名。
5. 规划 PostgreSQL、Redis、对象存储、日志、备份、监控。

验收标准：

1. H5 域名可公网访问。
2. 小程序合法域名配置指向 API HTTPS 域名。
3. 图片、凭证、打款文件不存本地磁盘。
4. 数据库有备份和恢复方案。

### 卡 8.3：多端上线验收

范围：

1. 小程序店铺真实下单。
2. H5 店铺浏览器下单。
3. 一级给二级设置转供价。
4. 二级完成用户订单。
5. 退款、追扣、结算按供货链拆分。
6. 支付渠道 mock/真实切换有开关。

验收标准：

1. 小程序和 H5 共用同一后端数据。
2. 二级不能创建三级。
3. 未授权一级不能创建二级。
4. 未审核或未缴保证金二级不能销售。
5. 审计日志覆盖渠道授权、转供价调整、退款裁决、结算和打款。
