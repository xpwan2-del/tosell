# H5 与三级以内渠道供货计划

本文保留为三级供货专题文档。当前 P0 是 H5 独立虚拟商品小店，不再以微信小程序作为交付目标。

## 1. 产品边界

1. H5 店铺可在微信外浏览器独立打开。
2. 平台后台、商户后台和 H5 前台共用同一套数据库/API。
3. 平台可以开设平台自营店直接销售。
4. 平台也可以向一级商户供货，一级向二级供货，二级向三级供货。
5. 三级不得继续创建四级。
6. 支付账号未开通前，P0 做商户收款配置绑定、审核和确认收款，不做生产 mock 支付。
7. 熟人或内部渠道可由平台后台手工创建一级商户和店铺，并生成初始账号密码交付；陌生商户必须通过入驻申请页输入邀请码注册。
8. 保证金确认是销售和代理商品的硬门槛，未确认保证金前不能销售、不能选品上架、不能代理平台或上级商品、不能配置转供价。

## 2. 禁止边界

1. 四级渠道或更多层级。
2. 下级返佣、推广返佣、按比例返利。
3. 团队奖、团队业绩、平级奖、领导奖。
4. 邀请奖励、拉人奖励、招募人数奖励。
5. 代理等级、团队等级、按等级差异返利。
6. 以招募人数、团队订单、团队 GMV 作为收益依据。

推荐术语：一级商户、二级商户、三级商户、渠道授权、平台供货价、一级转供价、二级转供价、最终销售价、供货价差、销售差价、供货链快照。

## 3. 商品可见与价格隔离

核心规则：

1. 下级可以看到上级开放给自己的商品资料，用于选品和上架销售。
2. 隔离的是越级供货价，不是商品本身。
3. 平台给一级的供货价只对平台和对应一级可见。
4. 二级只看到一级给自己的一级转供价，不能看到平台给一级的供货价。
5. 三级只看到二级给自己的二级转供价，不能看到一级转供价和平台供货价。
6. 买家只看到最终销售价。
7. API 响应、后台表格、导出、浏览器网络响应和订单列表都必须执行该规则。

示例：

```text
平台给一级供货价：100.00，仅平台和一级可见
一级给二级转供价：110.00，仅平台、一级、对应二级可按权限查看
二级给三级转供价：130.00，仅平台、二级、对应三级可按权限查看
三级最终售价：150.00，买家可见
```

## 4. 邀请码归属

1. 平台后台可以手工创建一级商户和店铺，用于熟人或内部渠道；手工创建必须记录创建人、初始账号交付状态和审计日志。
2. 后台手工创建入口只能创建一级商户，不能跳过邀请码关系创建二级或三级。
3. 平台邀请码注册一级商户。
4. 一级邀请码注册该一级名下二级商户。
5. 二级邀请码注册该二级名下三级商户。
6. 三级不得生成邀请码。
7. 邀请码不产生奖励、返佣或拉人收益。

## 4.1 保证金硬门槛

1. 商户审核通过或平台后台手工创建后，默认进入待缴保证金状态。
2. 平台后台可以人工确认保证金缴纳状态，用于线下收款、熟人渠道或财务核对场景。
3. 人工确认保证金必须记录金额、凭证、操作人、确认时间、备注和审计日志。
4. 保证金未确认前，商户可以登录后台维护基础资料和店铺装修草稿，但不能正式营业。
5. 保证金未确认前，后端必须拒绝销售、选品上架、代理平台商品、代理上级商品、配置转供价和创建可支付订单。
6. 保证金门槛必须在 API、后台菜单、前台可购买状态和测试用例中同时体现。

## 5. 后台与数据库要求

必须落库：

1. 商户、店铺、邀请码、渠道关系。
2. 商户账号、手工创建来源、初始账号交付状态、保证金账户、保证金确认记录。
3. 平台商品、商户商品、自有商品审核。
4. 转供商品和转供价。
5. 商品图片、详情、库存、卡密。
6. 收款配置和支付记录快照。
7. 订单、供货链快照、售后、供货清算、追扣、审计。

禁止硬编码：

1. 商品内容、图片、价格、库存、卡密。
2. 店铺、商户、渠道关系、客服二维码。
3. 收款码、支付链接、mock 支付按钮。
4. 后台固定测试 ID 快捷操作。

## 6. 供货清算规则

1. 一级差价 = 一级转供价 - 平台供货价。
2. 二级差价 = 二级转供价 - 一级转供价。
3. 三级销售差价 = 最终售价 - 二级转供价。
4. 若订单只到二级销售，则二级销售差价 = 最终售价 - 一级转供价。
5. 订单必须固化供货链快照，历史订单不受后续调价影响。
6. 多级订单清算明细使用 `order_id + clearing_role` 唯一。
7. 商户使用自己的收款码/链接收款，P0 不做平台代收后 T+1 给商户打款；系统只生成供货应付、服务费、价差和追扣对账。

## 7. 验收标准

1. H5 店铺能在浏览器独立打开。
2. 平台自营店可完成下单、履约、退款和对账。
3. 平台后台可手工创建一级商户和店铺，生成账号密码，状态进入待保证金确认，并写审计。
4. 未确认保证金的一级、二级、三级均不能销售、选品上架、代理上级商品或配置转供价。
5. 一级保证金确认后可选择平台商品并上架。
6. 一级可开放商品给二级并设置一级转供价。
7. 二级保证金确认后能选品上架，但看不到平台给一级的供货价。
8. 二级可开放商品给三级并设置二级转供价。
9. 三级保证金确认后能选品上架，但看不到一级转供价和平台供货价。
10. 三级不可创建四级。
11. 后台和 H5 前台全部从数据库/API 读取真实数据。
12. 生产代码不出现固定测试店铺、固定测试商品、固定收款码或固定 mock 支付。
