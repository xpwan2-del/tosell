# 历史支付开通与收款配置对接流程

状态：历史参考，已被 `docs/18-payment-binding-and-voucher-removal-plan.md` 替代。后续开发、测试和 PM 派工以 `docs/18` 为准：后台支持支付宝商户、腾讯/微信商户、e支付、个人支付宝四种收款方式；前三种走接口回调/查单自动确认，个人支付宝只走人工确认；H5 删除上传付款截图主流程。

## 1. 已废止的旧口径

以下内容只保留作历史来源，不再作为开发依据：

1. 支付宝个人收款码。
2. 支付宝商户收款码或跳转链接。
3. 微信个人收款码。
4. 微信商户收款码或跳转链接。

废止原因：

1. “微信个人码/支付宝个人码”不能当作官方自动回调能力。
2. “上传付款截图”不能作为正式支付确认机制。
3. 当前正式目标已经改为后台收款绑定中心、接口回调/查单、异常订单和人工确认分离。

## 2. 当前有效方案

当前有效方案见 `docs/18-payment-binding-and-voucher-removal-plan.md`，摘要如下：

1. 支付宝商户：后台配置 AppID、应用私钥、支付宝公钥、支付产品、回调地址；回调/查单确认后自动更新订单。
2. 腾讯/微信商户：后台配置商户号、AppID、API v3 密钥、证书/私钥、回调地址；回调/查单确认后自动更新订单。
3. e支付：后台配置服务商、商户号、网关、密钥、验签方式、回调地址；只有具备可靠回调/查单时才可自动确认。
4. 个人支付宝：后台配置账号、二维码、说明；只进入待确认收款，由有权限账号人工确认。

## 3. 后续真实支付需要准备

支付宝商户：

1. 支付宝开放平台应用。
2. APPID。
3. 商户主体资料。
4. 应用公钥和支付宝公钥。
5. 手机网站支付或当面付等签约产品。
6. 支付通知和退款通知 HTTPS 回调域名。

腾讯/微信商户：

1. 微信支付商户号。
2. 商户平台超级管理员账号。
3. API v3 密钥。
4. 商户 API 证书和证书序列号。
5. 支付通知和退款通知 HTTPS 回调域名。
6. H5 或 Native/JSAPI 等对应产品权限。

服务器：

1. 可公网访问的 HTTPS API 域名。
2. H5 店铺域名。
3. 后台域名。
4. PostgreSQL、对象存储、日志、备份和监控。
5. 安全保存密钥和证书的环境变量或密钥管理服务。

e支付：

1. 服务商接口文档。
2. 商户号、应用 ID 或渠道 ID。
3. 下单接口、查单接口和回调地址配置方式。
4. 签名/验签规则。
5. 生产和测试环境网关。

## 4. 技术接入原则

1. 前端不得决定支付金额。
2. 支付金额必须等于后端订单应付金额。
3. 支付回调必须验签、校验金额、写原始 payload 并幂等处理。
4. 支付结果不得只依赖前端跳转。
5. 退款回调必须验签并幂等处理。
6. 真实支付接入后仍保留商户收款配置模型，不能把所有收款写死到平台账号。
7. 前端支付返回页不等于支付成功，必须以后端回调、主动查单或人工确认结果为准。

## 5. 不能做的事

1. 不能在生产环境启用 mock 支付。
2. 不能把 API 密钥、私钥、证书提交到 git。
3. 不能把收款码或支付链接写死在前端。
4. 不能让前端决定订单金额。
5. 不能跳过回调验签。
6. 不能把上传付款截图作为正式支付确认机制。
7. 不能用监听通知、抓账单、模拟登录方式把个人码伪装成自动支付。
