# V2 支付开通与对接流程

本项目 V2 先完成非支付经营能力，微信支付账号开通后再接入真实支付。当前代码保留 mock 支付用于本地闭环，生产环境必须关闭 mock。

## 需要先准备

1. 已认证微信小程序 AppID。
2. 微信支付商户号 MCH_ID。
3. 商户平台超级管理员账号。
4. 结算银行卡和主体资质资料。
5. 可公网访问的 HTTPS API 域名。
6. 服务端安全保存密钥和证书的环境。

## 在微信侧操作

1. 登录微信公众平台，确认小程序已完成主体认证。
2. 登录微信支付商户平台，申请或绑定商户号。
3. 在产品中心开通 JSAPI 支付。
4. 在 AppID 账号管理里绑定当前小程序 AppID。
5. 设置 API v3 密钥。
6. 下载商户 API 证书，记录证书序列号。
7. 配置支付通知 URL 和退款通知 URL。
8. 确认商户平台结算账户、费率、交易权限已经生效。

## 需要交给技术配置

```bash
WECHAT_APP_ID=""
WECHAT_MCH_ID=""
WECHAT_PAY_API_KEY=""
WECHAT_PAY_CERT_SERIAL_NO=""
WECHAT_PAY_PRIVATE_KEY_PATH=""
WECHAT_PAY_PLATFORM_CERT_PATH=""
WECHAT_PAY_NOTIFY_URL=""
WECHAT_REFUND_NOTIFY_URL=""
MOCK_PAYMENT_ENABLED="false"
```

## 技术接入步骤

1. 用户下单后由服务端调用微信 JSAPI 下单。
2. 服务端返回小程序调起支付需要的 `timeStamp`、`nonceStr`、`package`、`signType`、`paySign`。
3. 小程序调用 `wx.requestPayment`。
4. 微信支付回调到服务端，服务端必须校验签名、证书序列号、时间戳和金额。
5. 回调通过后落库：支付单、回调原文、订单状态、账务流水、审计日志。
6. 售后审批通过后服务端发起微信退款。
7. 微信退款回调到服务端，服务端验签并更新退款单、追扣、结算冻结或保证金扣减。
8. 每日导入微信账单，与平台订单、退款、服务费、结算单做对账。

## 上线验收

1. 真实 1 分钱下单成功。
2. 支付回调验签成功，重复回调不重复记账。
3. 真实退款成功。
4. 退款回调验签成功，重复回调不重复追扣。
5. 已履约订单进入 T+1 结算。
6. 退款中、风控中、投诉中订单不进入结算。
7. 对账日报金额一致。

## 不能做的事

1. 不能在生产环境启用 mock 支付。
2. 不能把 API v3 密钥、私钥、证书提交到 git。
3. 不能让前端决定支付金额。
4. 不能跳过回调验签。
5. 不能把支付结果只依赖小程序前端返回。
