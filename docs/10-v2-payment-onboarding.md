# 支付开通与收款通道对接流程

状态：历史 V2 支付文档已废止，本文已改为 P0 后续真实支付接入参考，不作为当前开发优先级依据。当前 P0 必须先完成“商户收款通道绑定、审核、H5 展示、订单收款确认、自动/手工履约”。

## 1. 当前 P0 收款口径

P0 支持每个商户绑定自己的收款通道：

1. 支付宝个人收款码。
2. 支付宝商户收款码或跳转链接。
3. 微信个人收款码。
4. 微信商户收款码或跳转链接。

要求：

1. 收款通道必须落库并经过平台审核。
2. H5 支付页只展示当前店铺已启用通道。
3. 订单支付记录必须保存收款通道快照。
4. 收款码、账号名称、跳转链接不得硬编码。
5. mock 支付只能用于开发测试，生产环境不得展示。

## 2. 后续真实支付需要准备

支付宝：

1. 支付宝开放平台应用。
2. APPID。
3. 商户主体资料。
4. 应用公钥和支付宝公钥。
5. 手机网站支付或当面付等签约产品。
6. 支付通知和退款通知 HTTPS 回调域名。

微信：

1. 微信支付商户号。
2. 商户平台超级管理员账号。
3. API v3 密钥。
4. 商户 API 证书和证书序列号。
5. 支付通知和退款通知 HTTPS 回调域名。
6. H5 或 Native/JSAPI 等对应产品权限。

服务器：

1. 可公网访问的 HTTPS API 域名。
2. H5 店铺域名。
3. 后台域名。
4. PostgreSQL、对象存储、日志、备份和监控。
5. 安全保存密钥和证书的环境变量或密钥管理服务。

## 3. 技术接入原则

1. 前端不得决定支付金额。
2. 支付金额必须等于后端订单应付金额。
3. 支付回调必须验签、校验金额、写原始 payload 并幂等处理。
4. 支付结果不得只依赖前端跳转。
5. 退款回调必须验签并幂等处理。
6. 真实支付接入后仍保留商户收款通道配置模型，不能把所有收款写死到平台账号。

## 4. 不能做的事

1. 不能在生产环境启用 mock 支付。
2. 不能把 API 密钥、私钥、证书提交到 git。
3. 不能把收款码或支付链接写死在前端。
4. 不能让前端决定订单金额。
5. 不能跳过回调验签。
