# V2 非支付经营增强交付说明

V2 在不依赖真实支付商户号的前提下，先完成平台经营、平台自营、渠道经营和履约运营能力。真实支付接入见 `docs/10-v2-payment-onboarding.md`。

## 当前已覆盖能力

1. 渠道店铺装修：主题色、头图、分享标题、商品分组。
2. 渠道经营看板：订单数、GMV、预估收益、退款率、商品数、未读消息。
3. 批量选品：渠道商可一次处理多个平台商品，并由后端统一校验最低售价。
4. 权益码池：后台导入虚拟权益码，平台商品可切换为码池自动履约模式。
5. 模拟自动履约：mock 支付成功后，如果商品使用权益码池且库存可用，自动发码并更新履约状态。
6. 消息通知：店铺装修、批量选品、自动履约、库存不足等事件可生成渠道通知。
7. 风险看板：退款率、风控冻结、低保证金、低库存、待处理售后。
8. 支付开通指引：后台/API 返回微信支付、H5 支付和支付宝 WAP 后续开通所需资料、环境变量和验收步骤。

## 当前新增规划口径

1. 平台自营店：平台可直接面向消费者销售，订单归属为 `platform_self_operated`。
2. 单层渠道：渠道商在自己店铺销售，赚取售价与平台供货价之间的差价。
3. 受控两级渠道：平台供货 -> 一级渠道商 -> 二级渠道商 -> 消费者。
4. 平台自营订单不生成渠道结算，进入平台收入、履约成本、支付通道费、退款和 ledger 对账。
5. 两级渠道订单按 `order_id + settlement_role` 区分一级和二级结算明细。
6. H5 与小程序共用同一后端商品、订单、履约、售后、退款、结算和风控。
7. 支付渠道按场景预留：微信小程序支付、微信内 H5 JSAPI、微信外 H5 支付、支付宝手机网站支付、mock。
8. 微信小程序内不设计支付宝支付入口。

## 仍不包含

1. 真实微信小程序支付。
2. 真实微信 JSAPI/H5 支付。
3. 真实支付宝手机网站支付。
4. 真实微信或支付宝退款。
5. 生产级微信登录和 H5 登录。
6. 生产级后台账号 RBAC。
7. 真实文件存储。

这些能力需要外部账号、证书、域名、备案和密钥后才能接入，不能用本地代码伪造为已完成。

## 验收口径

1. `npm test` 通过，包含 V1 与 V2 后端行为测试。
2. `npm run typecheck` 通过。
3. `npm run db:validate` 通过。
4. `npm --workspace @tosell/admin run build` 通过。
5. 前端静态检查不得出现返佣、团队奖、邀请奖励、代理等级、三级渠道等文案。
6. 用户端公开页面不得展示供货价、转供价、服务费、渠道差价、结算金额、冻结金额。
7. mock 支付不得绕过订单、履约、退款、结算、ledger 和对账逻辑。

## 业务边界

V2 继续坚持供货价差和自营销售模型，不做返佣分销。平台自营、单层渠道和受控两级渠道可以并存；二级渠道商不得继续创建三级渠道商。
