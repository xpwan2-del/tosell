# V2 非支付经营增强交付说明

V2 在不依赖微信支付商户号的前提下，先完成平台经营、代理经营和履约运营能力。真实支付接入见 `docs/10-v2-payment-onboarding.md`。

## 已覆盖能力

1. 代理店铺装修：主题色、头图、分享标题、商品分组。
2. 代理经营看板：订单数、GMV、预估收益、退款率、商品数、未读消息。
3. 批量选品：代理可一次处理多个平台商品，并由后端统一校验最低售价。
4. 权益码池：后台导入虚拟权益码，平台商品可切换为码池自动履约模式。
5. 模拟自动履约：mock 支付成功后，如果商品使用权益码池且库存可用，自动发码并更新履约状态。
6. 消息通知：店铺装修、批量选品、自动履约、库存不足等事件可生成代理通知。
7. 风险看板：退款率、风控冻结、低保证金、低库存、待处理售后。
8. 支付开通指引：后台/API 返回微信支付开通所需账号、证书、环境变量和验收步骤。

## 仍不包含

1. 真实微信支付。
2. 真实微信退款。
3. 生产级微信登录。
4. 生产级后台账号 RBAC。
5. 真实文件存储。

这些能力需要外部账号、证书、域名和密钥后才能接入，不能用本地代码伪造为已完成。

## 验收口径

1. `npm test` 通过，包含 V1 与 V2 后端行为测试。
2. `npm run typecheck` 通过。
3. `npm run db:validate` 通过。
4. `npm --workspace @tosell/admin run build` 通过。
5. 前端静态检查不得出现多级分销、返佣、团队奖、邀请奖励、代理等级等文案。
6. 用户端公开页面不得展示供货价、服务费、代理收益、结算金额、冻结金额。

## 业务边界

V2 继续保持 B2B2C 微商小店/渠道供货模式。代理收入来自售价与供货价之间的差价，扣除平台服务费后形成可结算收益，不做多级分销、下级返佣、团队奖、邀请奖励或代理等级返利。
