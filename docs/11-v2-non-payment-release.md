# 历史非支付增强说明

状态：历史参考，不作为 P0 开发依据。当前 P0 以 `docs/14-h5-production-realignment-plan.md`、`docs/03-development-cards.md`、`docs/07-admin-backend-database-plan.md`、`docs/09-api-testing-release-plan.md` 为准。

## 保留价值

以下能力仍然有价值，但必须按当前 H5 生产口径重新实现：

1. 店铺装修。
2. 商户经营看板。
3. 批量选品。
4. 权益码池和自动发码。
5. 消息通知。
6. 风险看板。
7. 支付开通指引。

## 当前口径

1. 不做小程序 P0。
2. 不使用生产 mock 支付。
3. 不使用前端固定 demo store。
4. 不允许后台固定 `shop-1/agent-1/prod-1` 快捷操作。
5. 商品、店铺、客服、收款码、库存、卡密、订单都必须来自数据库/API。
6. 二级能看到一级开放给自己的商品和一级转供价，但不能看到平台给一级的供货价。
7. 三级能看到二级开放给自己的商品和二级转供价，但不能看到一级转供价和平台供货价。

## 不再作为依据的内容

1. 小程序端交付。
2. 小程序支付。
3. H5 与小程序共用后端作为 P0 验收。
4. mock 支付灰度作为生产上线验收。
