# 业务规则与账务规则

## 1. 商业模式

项目采用 B2B2C 微商小店/受控渠道供货模式。

平台向渠道商提供虚拟商品供货和履约能力。渠道商在自己的小店中销售商品，用户看到的是渠道店铺和渠道售价。单层店铺收益来自售价与平台供货价之间的差价，平台每笔订单收取千五服务费。

禁止将本项目设计为多级分销、下级返佣或团队业绩奖励系统。后续允许受控两级渠道供货：平台 -> 一级渠道商 -> 二级渠道商 -> 消费者。一级渠道商收益来自一级转供价与平台供货价之间的供货价差，二级渠道商收益来自最终售价与一级转供价之间的销售差价。

## 2. 商品规则

### 2.1 平台商品

平台商品由平台创建和维护。

每个商品至少包含：

1. 商品名称。
2. 商品分类。
3. 商品详情。
4. 权益说明。
5. 平台供货价。
6. 最低销售价。
7. 建议销售价。
8. 履约方式。
9. 售后规则。
10. 上架状态。

平台商品由平台负责主要履约和供货质量。

### 2.2 代理自有商品

代理可以上传自有虚拟商品，但必须经过平台审核后才能售卖。

代理自有商品建议规则：

1. 代理负责主要履约。
2. 代理负责主要售后。
3. 平台保留审核、下架、冻结和仲裁权。
4. 平台按订单收取千五服务费。
5. 代理自有商品造成用户投诉或损失时，优先从代理收益或保证金中扣减。

第一版支持代理提交自有虚拟商品，所有自有商品必须经过平台审核后才可在该代理店铺售卖。平台可以基于风控策略限制部分代理使用该能力，但系统能力需要具备。

## 3. 定价规则

代理可以自由设置商品销售价，但不得低于平台配置的最低销售价。

后台字段建议：

1. 平台供货价：平台与代理之间的结算基础。
2. 最低销售价：代理售价不得低于该价格。
3. 建议销售价：平台给代理的推荐销售价格。
4. 代理销售价：用户实际看到和支付的价格。
5. 一级转供价：一级渠道商给二级渠道商的供货价。
6. 二级最终售价：消费者实际看到和支付的价格。

价格校验必须以后端为准，前端提示只能作为辅助。

两级渠道价格校验：

1. 一级转供价不得低于平台供货价。
2. 二级最终售价不得低于一级转供价。
3. 二级最终售价仍需满足平台配置的最低销售价或价格风控规则。
4. 平台供货价、一级转供价、二级最终售价必须写入订单供货链快照。

## 4. 订单归属规则

用户通过代理店铺链接、二维码或分享路径进入小程序后，下单订单归属于该代理店铺。

建议第一版采用以下归属规则：

1. 用户从代理店铺进入时，当前会话默认绑定该代理店铺。
2. 用户在该店铺内下单，订单归属该代理。
3. 若用户直接进入平台首页，默认不归属代理，除非仍在有效绑定期内。
4. 是否设置 30 天店铺绑定期需要业务方最终确认。

订单归属必须在下单时固化，后续不因商品价格变化、代理状态变化或用户再次进入其他入口而改变历史订单归属。

## 5. 服务费规则

平台每笔订单收取千五服务费，即 0.5%。

建议计算口径：

```text
服务费 = 用户实付金额 * 0.5%
```

例如：

```text
用户实付金额：150.00
平台供货价：100.00
服务费：150.00 * 0.5% = 0.75
代理预计收益：150.00 - 100.00 - 0.75 = 49.25
```

服务费退款口径：

1. 未履约全额退款：服务费退回，不计入平台服务费收入。
2. 平台责任导致的退款：服务费退回，由平台承担。
3. 代理责任导致的退款：服务费默认不退；如平台仲裁决定退回，计入代理责任金额，可从代理未结算收益或保证金追扣。
4. 用户原因退款：服务费默认不退。
5. 部分退款：服务费按退款金额占用户实付金额的比例退回；责任归属决定由平台承担还是代理承担。

## 6. 收益规则

平台商品订单的代理收益建议计算：

```text
代理收益 = 用户实付金额 - 平台供货价 - 平台服务费
```

两级渠道平台商品订单的建议计算：

```text
一级渠道差价 = 一级转供价 - 平台供货价
二级销售差价 = 用户实付金额 - 一级转供价 - 平台服务费承担项
```

示例：

```text
平台供货价：100.00
一级转供价：110.00
用户实付金额：150.00
一级渠道差价：10.00
二级销售差价：40.00，服务费按配置扣减
```

代理自有商品订单的代理收益建议计算：

```text
代理收益 = 用户实付金额 - 平台服务费
```

若代理自有商品未来存在平台代履约成本，则需要单独增加成本项。

所有金额必须在订单创建或支付成功时生成快照，避免后续价格调整影响历史订单。

两级渠道订单必须额外固化：一级渠道商、二级渠道商、一级店铺、二级店铺、平台供货价、一级转供价、最终售价、一级渠道差价、二级销售差价、服务费承担规则和渠道授权状态。

## 7. 保证金规则

代理入驻需要缴纳保证金。保证金用于覆盖退款追扣、违规扣罚、投诉赔付、恶意刷单、虚假宣传和售后失责等风险。

建议规则：

1. 未缴纳保证金的代理不能正式开店销售。
2. 保证金不足时，平台可以限制结算、暂停人工打款、限制上架或暂停店铺。
3. 保证金扣减必须有原因、金额、关联订单或关联事件。
4. 代理退出时，如果没有未完成订单、退款、投诉、冻结或追扣，经过观察期后退还保证金。
5. 观察期建议为 15-30 天。

保证金金额、缴纳方式和退还周期需要业务方最终确认。

## 8. 履约规则

虚拟商品履约可以是自动发放或人工发放。

履约状态建议：

1. 未履约。
2. 履约中。
3. 履约成功。
4. 履约失败。
5. 已补发。
6. 已撤销。

履约要求：

1. 每次发放必须记录发放凭证。
2. 发放失败必须支持重试或人工补发。
3. 必须避免重复发放。
4. 退款后如果权益可撤销，应记录撤销状态。
5. 用户端应能看到权益发放结果。

## 9. 退款规则

退款按用户实际支付金额处理，而不是只退供货价或只退代理差价。

### 9.1 未履约退款

虚拟权益未发放、未激活、未使用时，建议支持全额退款。

全额退款时：

1. 用户退回实际支付金额。
2. 平台退回供货价部分。
3. 代理退回差价部分。
4. 未履约全额退款时，服务费建议退回。

### 9.2 已履约未使用退款

如果权益已发放但可证明未使用、未下载、未激活，可以支持全额退款，并撤销权益。

必须具备权益状态判断能力，否则建议进入人工审核。

### 9.3 已使用或已激活退款

已使用、已激活、已下载或已消费的虚拟权益，默认不支持无理由退款。

但以下场景可以人工仲裁：

1. 平台履约失败。
2. 商品描述不符。
3. 代理虚假宣传。
4. 用户无法正常使用且责任不在用户。

### 9.4 责任归属

退款责任分为：

1. `platform`：平台责任，供货失败、权益不可用、平台商品描述错误。
2. `agent`：单层代理责任，虚假宣传、私自承诺、售后失责、自有商品问题。
3. `first_tier`：两级渠道中的一级渠道商责任，例如转供商品配置错误、一级承诺或一级售后失责。
4. `second_tier`：两级渠道中的二级渠道商责任，例如最终销售宣传、客服承诺、二级自有商品或售后失责。
5. `user`：用户原因，误购、不想要、已使用后主观退款。
6. `mixed`：混合责任，平台、一级、二级或用户中多个主体共同承担。

系统需要在售后单中记录责任归属。
单层订单可以继续使用 `agent`；两级渠道订单不得只写 `agent`，必须明确映射到 `first_tier` 或 `second_tier`，混合责任时记录各主体承担金额。

### 9.5 退款拆账矩阵

退款按用户实付金额发起。售后单必须记录退款总额、平台承担金额、代理承担金额、服务费处理方式和责任归属。

| 场景 | 用户退款 | 平台承担 | 代理承担 | 服务费处理 |
| --- | --- | --- | --- | --- |
| 未履约全额退款 | 用户实付金额 | 平台供货价部分 | 代理差价部分 | 退回 |
| 平台责任退款 | 平台仲裁金额 | 平台供货价及平台责任赔付 | 原则上不承担，除非有代理共同责任 | 退回，由平台承担 |
| 代理责任退款 | 平台仲裁金额 | 正常供货价不承担额外损失 | 代理差价及代理责任赔付 | 默认不退；若退回，由代理承担 |
| 用户原因退款 | 平台审核金额 | 按实际撤销权益情况处理 | 按实际退款差价处理 | 默认不退 |
| 混合责任退款 | 平台仲裁金额 | 平台录入承担金额 | 代理录入承担金额 | 平台在售后单中指定承担方 |

如果订单未结算，代理承担金额从该订单待结算收益中扣减；如果订单已结算，进入追扣流程。

两级渠道退款必须记录平台、一级、二级、用户或混合责任。售后单需要记录用户退款金额、平台承担金额、一级承担金额、二级承担金额、服务费退还金额和服务费承担方。未结算订单直接冻结并扣减待结算收益；已结算订单分别生成一级或二级追扣。

两级渠道退款矩阵：

| 场景 | 用户退款 | 平台承担 | 一级承担 | 二级承担 | 服务费处理 |
| --- | --- | --- | --- | --- | --- |
| 未履约全额退款 | 用户实付金额 | 平台供货价部分 | 一级渠道差价 | 二级销售差价 | 默认退回，按配置指定承担方 |
| 平台责任退款 | 平台仲裁金额 | 平台供货价及平台责任赔付 | 原则上不承担 | 原则上不承担 | 退回，由平台承担 |
| 一级责任退款 | 平台仲裁金额 | 正常供货价不承担额外损失 | 一级渠道差价及责任赔付 | 按实际责任处理 | 默认不退；若退回，由一级承担或按仲裁分摊 |
| 二级责任退款 | 平台仲裁金额 | 正常供货价不承担额外损失 | 原则上不承担 | 二级销售差价及责任赔付 | 默认不退；若退回，由二级承担 |
| 混合责任退款 | 平台仲裁金额 | 平台录入承担金额 | 一级录入承担金额 | 二级录入承担金额 | 平台在售后单中指定承担方 |

## 10. 结算规则

结算周期暂定 T+1。

建议口径：

```text
T+1 从履约成功时间开始计算，而不是从支付成功时间开始计算。
```

原因：虚拟商品如果支付成功但履约失败，不应进入结算。

订单进入可结算前必须满足：

1. 支付成功。
2. 履约成功。
3. 未退款。
4. 未处于退款中。
5. 未被风控冻结。
6. 未被投诉冻结。

结算状态建议：

1. 待结算。
2. 冻结中。
3. 可结算。
4. 结算中。
5. 已结算。
6. 已追扣。

第一版不建设代理自助提现能力。代理收益通过平台生成结算单后人工打款；风控时限制结算或暂停人工打款。

两级渠道结算：

1. 一级和二级分别生成结算明细。
2. 同一订单不能重复结算。
3. 结算明细必须包含平台供货价、一级转供价、最终售价、服务费、一级差价、二级差价。
4. 手工打款流程仍为结算单、导出、人工打款、回填凭证。
5. 追扣顺序分别适用于一级和二级：未结算收益 -> 待打款收益 -> 保证金。

## 11. 追扣规则

如果订单已结算后发生退款、投诉赔付或违规扣罚，平台需要从代理后续收益或保证金中追扣。

追扣顺序建议：

1. 优先从未结算收益中扣。
2. 不足部分从待打款收益中扣。
3. 仍不足时从保证金中扣。
4. 保证金不足时限制店铺销售、限制结算或暂停人工打款。

所有追扣都必须可追溯。

## 12. 风控规则

第一版至少支持人工风控：

1. 冻结订单。
2. 解冻订单。
3. 冻结店铺。
4. 禁用店铺。
5. 限制结算或暂停人工打款。
6. 商品下架。
7. 保证金扣减。

建议触发风控的情况：

1. 退款率过高。
2. 投诉率过高。
3. 代理价格异常高。
4. 代理宣传违规。
5. 履约失败过多。
6. 保证金不足。
7. 疑似刷单。

## 13. 数据隔离规则

代理只能查看自己的店铺、商品、订单、用户购买记录、收益和售后。

代理不能查看：

1. 其他代理的订单。
2. 其他代理的客户。
3. 平台全局经营数据。
4. 平台供货成本之外的内部财务数据。

平台后台按照角色权限控制数据访问。
