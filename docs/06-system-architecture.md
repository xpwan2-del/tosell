# 系统整体架构

本文是 V1 开发依据文档，描述微信小程序虚拟商品 B2B2C 微商小店/渠道供货平台的整体架构、端划分、后端领域、数据流和边界。后续开发必须以数据库、后端账务、状态机、权限隔离和审计为核心，前端只负责展示、录入和辅助提示。

## 1. 系统定位

本项目不是前端小程序项目，而是交易、履约、售后、结算、风控、审计一体的财务型业务系统。

V1 目标是跑通以下真实闭环：

代理入驻审核 -> 缴纳保证金 -> 开通独立店铺 -> 选择平台商品或提交自有商品审核 -> 代理定价 -> 用户从代理店铺下单支付 -> 虚拟权益履约 -> 售后退款 -> T+1 结算单 -> 人工打款 -> 对账审计。

系统必须坚持：

1. 代理审核通过并缴纳保证金后才能正式销售。
2. 每个代理拥有独立店铺、店铺链接、商品列表、客服微信、订单、收益和结算记录。
3. 平台商品有供货价、最低销售价、建议销售价、履约规则和售后规则。
4. 代理可自由定价，但后端必须校验售价不得低于最低销售价。
5. 代理收益来自售价与平台供货价之间的差价，扣除平台 0.5% 服务费后形成代理预计收益，不是返佣。
6. 代理自有商品必须平台审核通过后才能售卖。
7. 订单、退款、结算、保证金和追扣必须可追溯。
8. V1 只做结算单加人工打款，不做代理自助提现。

## 2. 禁止边界

系统不得设计、实现或预留以下能力：

1. 多级分销。
2. 下级返佣。
3. 团队业绩奖、团队排名奖、平级奖、领导奖。
4. 邀请奖励、拉新代理奖励。
5. 代理等级体系。
6. 下级订单统计、下级客户统计。
7. 返佣比例、团队业绩、代理层级等模型。

数据库、API、页面、导出报表和运营后台中不得出现 `parent_agent_id`、`inviter_agent_id`、`agent_level`、`team_id`、`commission_rate`、`downline_order`、`team_performance` 等分销字段或同义字段。

分享路径只能用于确定用户进入的 `shop_id/agent_id` 和订单归属，不能用于发展下级或计算奖励。

## 3. 角色与端划分

### 3.1 普通用户小程序

面向购买用户，包含代理店铺首页、商品列表、商品详情、下单支付、支付结果、订单中心、权益查看、售后申请和客服微信展示。

用户端只能展示售价、实付金额、订单状态、履约状态、权益信息和售后信息。用户端不得展示平台供货价、服务费、代理收益、结算金额、冻结金额和后台风控细节。

### 3.2 代理中心

V1 可做小程序内代理中心或 Web 代理中心，优先保证与后端合同清晰。包含入驻申请、审核状态、保证金状态、店铺配置、平台商品选品、代理定价、自有商品提交、订单收益、结算记录、冻结记录、追扣记录。

代理端可以展示平台商品供货价、最低价、建议价、代理售价、预估收益、订单收益和结算状态，但所有金额以后端返回为准。代理端不能退款裁决、不能确认结算、不能人工打款、不能扣减保证金、不能解除风控、不能查看其他代理数据。

### 3.3 平台后台

面向运营、财务和管理员，包含代理审核、保证金管理、平台商品库、代理商品审核、订单管理、履约管理、售后仲裁、结算管理、人工打款、风控冻结、投诉处理、审计日志和基础看板。

后台必须区分运营、财务、管理员权限。运营不能执行人工打款和结算确认；财务不能修改商品价格和商品审核；管理员负责 RBAC、高风险配置和最终权限管理。

### 3.4 后端服务

后端是金额、状态、权限、结算、账务和审计的唯一权威。前端提交的价格、服务费、收益、退款金额、结算金额只能作为请求参数或展示辅助，不能作为最终依据。

### 3.5 数据库与账务

数据库是财务追溯核心。订单必须固化快照；退款、结算、追扣、保证金扣减必须写流水；账务流水只能追加，不能原地覆盖金额。

## 4. 核心领域模块

1. Auth/RBAC：微信登录、后台账号、角色权限、菜单权限、数据权限、敏感操作授权。
2. Agent/Shop：代理入驻申请、审核、保证金状态联动、店铺资料、客服微信、冻结、禁用。
3. Product/AgentProduct：平台商品库、代理选品、代理自有商品提交审核、上下架、风险下架。
4. Pricing：最低限价校验、服务费计算、代理收益计算、金额取整、订单金额快照。
5. Order：订单创建、店铺归属固化、订单状态机、超时关闭、订单查询与代理隔离。
6. PaymentCallback：微信支付单、支付回调验签、金额校验、幂等入账、支付成功触发履约。
7. Fulfillment：自动/人工履约、履约尝试、权益发放、失败重试、补发、撤销、重复发放拦截。
8. AfterSale/Refund：售后申请、冻结结算、责任归属、退款审批、退款回调、退款拆账。
9. Settlement：T+1 可结算筛选、结算单生成、结算明细唯一约束、人工打款确认。
10. Deposit：保证金账户、缴纳确认、扣减、冻结、退还、余额不足限制。
11. RiskFreeze：订单冻结、店铺冻结、限制结算、禁用、解冻后重新判断结算资格。
12. Ledger/Audit：不可变账务流水、审计日志、冲正流水、敏感操作留痕。
13. Reconciliation/Export：订单、支付、退款、结算、人工打款、保证金、ledger 对账导出。

## 5. 主数据流

### 5.1 代理入驻与店铺开通

1. 代理提交入驻申请和客服微信。
2. 运营审核申请。
3. 审核通过后进入待缴保证金状态。
4. 财务确认保证金收款，写保证金交易和账务流水。
5. 保证金满足要求后开通代理和店铺。
6. 代理配置店铺资料和客服微信。

### 5.2 商品与定价

1. 平台创建平台商品，配置供货价、最低销售价、建议销售价、履约规则和售后规则。
2. 代理选择平台商品上架并设置销售价。
3. 后端校验销售价不得低于最低销售价。
4. 代理提交自有商品时，必须先进入平台审核流程。
5. 自有商品审核通过后才可上架售卖。

### 5.3 下单支付与订单快照

1. 用户通过代理店铺链接进入小程序。
2. 用户选择商品并创建订单。
3. 后端校验代理、店铺、保证金、商品、售价和风控状态。
4. 后端计算用户实付金额、供货价、服务费和代理预计收益。
5. 后端写入订单、订单项、金额快照、商品快照、店铺快照和规则快照。
6. 支付回调验签、校验金额、幂等处理。
7. 支付成功后写支付记录和账务流水，并触发履约。

### 5.4 履约与权益

1. 支付成功后创建履约任务。
2. 自动履约或人工履约写入履约尝试记录。
3. 履约成功生成用户权益凭证。
4. 履约失败记录失败原因，订单不得进入可结算。
5. 补发、重试、撤销都必须有凭证和审计。

### 5.5 售后退款

1. 用户提交售后申请。
2. 订单进入退款中或售后冻结状态，排除结算。
3. 代理先处理，必要时平台介入。
4. 后台录入责任归属、退款金额、平台承担、代理承担、服务费处理。
5. 退款回调幂等更新结果。
6. 未结算订单扣减待结算收益；已结算订单生成追扣。

### 5.6 T+1 结算与人工打款

1. 定时任务按代理和窗口筛选可结算订单。
2. 条件包括支付成功、履约成功、超过 T+1、无退款中、无投诉中、无风控冻结。
3. `settlement_items.order_id` 必须唯一，防止同一订单重复结算。
4. 财务确认结算单并导出人工打款文件。
5. 财务完成线下或人工打款后回填凭证。
6. 系统写人工打款记录、账务流水和审计日志。

### 5.7 风控与审计

1. 平台可冻结订单、店铺、商品或限制结算。
2. 冻结期间相关订单不得进入结算。
3. 解冻后重新按状态机判断是否可结算。
4. 退款裁决、保证金扣减、结算确认、人工打款、店铺禁用、商品风险下架、权限变更必须写审计日志。

## 6. 外部依赖

1. 微信小程序登录：获取用户身份和 openid。
2. 微信支付：支付单、支付回调、退款单、退款回调、渠道对账。
3. 小程序分享路径和二维码：携带店铺标识，只用于店铺归属。
4. 文件与图片存储：商品图片、客服二维码、售后凭证、打款凭证。
5. 虚拟权益履约接口：如 V1 接第三方或自动发放，需要履约回调和幂等处理；若 V1 人工发放，也必须记录发放凭证。

## 7. V1 必做

1. 用户端交易闭环。
2. 代理端入驻、保证金、店铺、选品、定价、订单收益、结算记录。
3. 平台后台审核、商品、订单、履约、售后、结算、风控、审计。
4. 后端订单快照、金额计算、支付/退款回调幂等、履约、售后、结算、追扣、保证金、权限、审计。
5. 数据库核心实体、金额字段、唯一约束、索引、账务流水、审计日志。
6. 灰度真实支付、真实退款、结算单、人工打款、追扣、保证金扣减、风控冻结和日报平账。

## 9. 默认口径与系统配置

以下参数仍需业务最终确认，但为避免开发口径悬空，V1 默认按以下配置实现，后续可做系统配置项：

1. 服务费率：`50 bps`，即用户实付金额的 0.5%。
2. 金额单位：统一使用分 `amount_cents`，禁止浮点。
3. 服务费取整：按分计算后四舍五入到分，订单快照固化实际服务费金额。
4. T+1 起算：默认按履约成功时间起算。
5. 追扣顺序：未结算收益 -> 待打款收益 -> 保证金；仍不足时限制结算、上架或店铺销售。
6. 结算方式：V1 生成结算单并人工打款，不做代理自助提现。
7. 店铺归属：订单创建时固化当前店铺归属；是否有 30 天绑定期仍待确认。
