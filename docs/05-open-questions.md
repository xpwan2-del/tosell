# 待补充业务参数

本文只记录仍需业务方补充的参数和口径。已经确认的规则不得再作为开放问题处理。

## 1. 已确认规则

1. 项目是微信小程序虚拟商品销售平台。
2. 商业模式是 B2B2C 微商小店/渠道供货，不做多级返佣分销。
3. 代理需要审核入驻。
4. 代理需要缴纳保证金后才能正式销售。
5. 代理拥有自己的店铺页、推广链接、客服微信和售后团队。
6. 代理赚取售价与平台供货价之间的差价，不是返佣。
7. 代理可以自由定价，但不得低于平台最低限价。
8. 平台每笔订单收取 0.5% 服务费。
9. 服务费按用户实付金额计算，并从订单差额中扣除后计入代理预计收益。
10. 代理可以提交自有虚拟商品，必须经过平台审核后才可售卖。
11. 退款按用户实付金额处理，平台退供货部分、代理退差价部分，并按责任承担损失。
12. 结算采用 T+1，但退款中、投诉中、风控中的订单必须冻结。
13. 第一版不做多级团队、下级返佣、团队奖、邀请奖励或代理等级体系。
14. 第一版不建设代理自助提现能力，采用结算单加人工打款。
15. 小程序不是 HTML，需要新增 H5 独立店铺才能在微信外浏览器打开。
16. 后续允许受控两级渠道供货：平台 -> 一级渠道商 -> 二级渠道商 -> 消费者。
17. 受控两级渠道收益来自供货价差，不是返佣；二级不得继续创建三级渠道。

## 2. 保证金参数

1. 保证金金额是多少？
2. 是否所有代理统一金额？
3. 是否根据商品类型或风险等级设置不同金额？
4. 保证金通过线上支付还是人工收款记录？
5. 保证金退还观察期是 15 天、30 天还是其他周期？
6. 保证金不足时，是限制结算、限制上架，还是直接暂停销售？

## 3. 服务费退款口径

已确认：平台每笔订单收取 0.5% 服务费，按用户实付金额计算。

仍需补充：

1. 平台责任退款时，服务费是否全部退回？
2. 代理责任退款时，服务费若退回是否计入代理承担金额？
3. 用户原因退款时，服务费是否默认不退？
4. 部分退款时，服务费是否按退款比例处理？

当前建议：未履约全额退款退服务费；部分退款按退款比例处理；责任退款由责任方承担服务费退回成本。

## 4. 结算参数

已确认：结算采用 T+1，退款中、投诉中、风控中的订单冻结。

仍需补充：

1. T+1 最终按支付成功时间算，还是按履约成功时间算？
2. 结算单是否需要财务二次确认？
3. 人工打款记录需要记录哪些字段？
4. 结算后发生退款时，追扣优先从未结算收益、待打款收益还是保证金扣？

当前建议：T+1 从履约成功时间算；第一版生成结算单后人工打款。

## 5. 店铺归属参数

1. 用户首次进入代理店铺后，是否绑定该店铺？
2. 绑定有效期是否为 30 天？
3. 用户从另一个代理店铺进入时，是否覆盖原绑定？
4. 用户直接进入平台首页购买时，是否归属最近绑定代理？

当前建议：订单归属在下单时固化；是否做 30 天绑定由业务最终确认。

## 6. 代理自有商品参数

已确认：代理可以提交自有商品，必须经过平台审核后才可售卖。

仍需补充：

1. 是否所有审核通过的代理都可以提交自有商品？
2. 是否需要根据保证金余额限制自有商品数量或销售额度？
3. 自有商品是否全部由代理履约？
4. 平台是否需要对自有商品设置最低价？
5. 自有商品是否需要额外类目或资料审核？

## 7. 履约方式参数

1. 虚拟权益是自动发放还是人工发放？
2. 是否需要对接第三方发放接口？
3. 权益是否支持撤销？
4. 权益是否有有效期？
5. 发放失败后自动重试几次？

## 8. 退款规则参数

1. 已发放但未使用的权益，是否支持全额退款？
2. 已使用或已激活的权益，是否默认不退？
3. 部分使用后是否支持部分退款？
4. 用户原因退款是否允许？
5. 平台和代理混合责任如何分摊？

## 9. 客服和售后参数

1. 用户默认先联系代理客服还是平台客服？
2. 代理多久不处理售后会自动升级到平台？
3. 平台介入后是否拥有最终裁决权？
4. 代理售后失责是否扣保证金？
5. 客服微信展示是否需要平台审核？

## 10. 价格风控参数

1. 是否需要最高销售价？
2. 是否需要价格异常提醒？
3. 代理是否可以做临时活动价？
4. 活动价是否同样不能低于最低限价？

## 11. 合规与小程序审核参数

1. 小程序主体和类目是否已经确认？
2. 虚拟商品支付能力是否可以开通？
3. 当前计划销售的虚拟商品是否符合类目要求？
4. 用户协议、隐私协议、代理协议是否需要法务审核？
5. 是否需要限制某些高风险虚拟商品类型？

## 12. H5 与服务器参数

1. H5 店铺域名使用哪个主体和哪个域名？
2. API 域名、H5 域名、后台域名是否同一备案主体？
3. 服务器优先选择腾讯云、阿里云还是其他国内云？
4. H5 用户登录优先使用手机号短信、网页登录，还是微信网页授权？
5. H5 是否需要和小程序用户通过手机号或 unionid 合并身份？
6. 对象存储使用腾讯云 COS、阿里云 OSS 还是其他方案？
7. 生产环境 PostgreSQL、Redis、备份、监控由谁维护？

## 13. 两级渠道参数

1. 一级渠道商申请开通二级渠道能力需要满足什么条件？
2. 一级渠道能力是否需要额外保证金或更高保证金？
3. 二级渠道商保证金金额是多少？
4. 一级可给二级开放全部商品，还是逐个商品授权？
5. 一级转供价是否必须不低于平台最低销售价，还是只需不低于平台供货价？
6. 二级最终售价是否需要最高价或价格异常提醒？
7. 服务费默认由二级销售差价承担，还是由一级/二级按配置分摊？
8. 一级责任、二级责任、混合责任的仲裁标准如何细化？

## 14. 多渠道支付参数

1. 真实支付优先接入微信小程序支付、微信 H5 支付还是支付宝手机网站支付？
2. 微信 H5 在微信内和微信外浏览器分别采用什么支付方式？
3. 支付宝应用、商户号和签约产品何时开通？
4. 是否需要统一支付渠道配置后台？
5. mock 支付在生产环境是否强制关闭并由部署检查拦截？
