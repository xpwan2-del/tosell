# 待补充业务参数

本文只记录仍需业务方补充的参数和口径。已经确认的规则不得再作为开放问题处理。

## 1. 已确认规则

1. 项目 P0 是 H5 独立虚拟商品小店系统，不再以微信小程序作为交付目标。
2. 系统包含平台后台、商户后台、多商户独立店铺前端、数据库、API、收款通道、卡密履约、售后和结算。
3. 商业模式是 B2B2C 微商小店/渠道供货，不做返佣、返利、团队奖、邀请奖励或代理等级。
4. 平台可以自营开店，也可以向一级商户供货。
5. 商户需要审核入驻并缴纳保证金后才能正式销售。
6. 商户拥有自己的店铺页、推广链接、客服二维码、收款通道、商品、订单和售后。
7. 商户可以自由定价，但不得低于平台最低限价和自己的上游供货价。
8. 平台每笔订单收取 0.5% 服务费；P0 默认按结算基准金额计算。平台券抵扣不降低供货价、转供价或各级价差结算基准，用户实付金额主要用于收款确认、退款上限和实收统计。
9. 商户可以提交自有虚拟商品，必须经过平台审核后才可售卖。
10. 自动卡密商品是否需要提取码由后台商品配置决定；提取码主要用于虚拟充值卡/兑换码。启用提取码时买家下单设置购买密码/提取码，确认收款后进入独立提取页只输入购买密码/提取码查看卡密。手工交付商品引导买家联系客服。
11. 买家邮箱为选填；填写邮箱时，确认收款发码后同步发送激活码/卡密到邮箱，未填写不影响下单和独立提取。邮箱发码是补充通道，不替代购买密码提取。
12. 商户使用自己的收款码/链接收款，P0 不做平台代收后 T+1 给商户打款结算。
13. 第一版不建设商户自助提现，采用供货应付、服务费、追扣和保证金对账。
14. 邀请码只用于商户归属：平台邀请码生成一级，一级邀请码生成二级，二级邀请码生成三级，三级不能生成四级。
15. 商品资料可以沿供货链开放给下级用于选品和销售。
16. 价格必须逐级隔离：二级不能看到平台给一级的供货价；三级不能看到一级转供价和平台供货价。
17. 逐级价格隔离必须在 API、后台表格、订单列表、导出和浏览器网络响应中执行。
18. 微信和支付宝只作为可选收款/支付通道；P0 先做商户收款通道绑定、审核和订单收款状态。
19. 平台后台可以手工创建一级商户和店铺，用于熟人或内部渠道，并生成初始账号密码交付；手工创建必须记录创建人、交付状态和审计日志。
20. 陌生商户必须通过商户入驻申请页输入邀请码注册；平台邀请码生成一级，一级邀请码生成二级，二级邀请码生成三级。
21. 保证金确认是销售和代理商品的硬门槛；未确认保证金前不能销售、不能选品上架、不能代理平台或上级商品、不能配置转供价、不能创建可支付订单。
22. 平台后台可以人工确认保证金缴纳状态，但必须记录金额、凭证、操作人、确认时间、备注和审计日志。

## 2. 保证金参数

1. 保证金金额是多少？
2. 一级、二级、三级是否统一保证金？
3. 是否根据商品类型或风险等级设置不同金额？
4. 保证金通过线上支付还是人工收款记录？
5. 保证金退还观察期是 15 天、30 天还是其他周期？
6. 保证金不足时，是限制结算、限制上架，还是直接暂停销售？

已确认：保证金未确认时必须限制销售、选品、上级商品代理、转供价配置和可支付订单创建。人工确认保证金允许存在，但必须审计。

## 3. 服务费与退款口径

已确认：平台每笔订单收取 0.5% 服务费；P0 默认 `settlement_basis_amount_cents = pre_coupon_sale_amount_cents`，服务费按结算基准金额计算。平台券属于平台补贴，不影响供货价、转供价和各级价差结算基准；`buyer_paid_amount_cents` 只用于收款确认、退款上限和实收统计，不直接替代价差结算基准。

仍需补充：

1. 平台责任退款时，服务费是否全部退回？
2. 商户责任退款时，服务费若退回是否计入责任商户承担金额？
3. 用户原因退款时，服务费是否默认不退？
4. 部分退款时，服务费是否按退款比例处理？
5. 多级供货订单中服务费默认由最终销售方承担，还是按配置在一级/二级/三级之间分摊？

当前建议：未履约全额退款退服务费；部分退款按退款比例处理；责任退款由责任方承担服务费退回成本。

## 4. 供货清算与对账参数

已确认：商户使用自己的收款码/链接收款，P0 不做平台代收后 T+1 给商户打款。系统需要做供货应付、服务费、追扣、保证金和对账；退款中、投诉中、风控中的订单冻结相关账单。

仍需补充：

1. 供货应付/服务费账单按日、按周还是按月生成？
2. 上游收款或线下补款的凭证需要记录哪些字段？
3. 已清算后发生退款时，追扣优先从未清算应付/应收、待确认清算金额还是保证金扣？
4. 平台服务费由商户主动补款、线下月结，还是未来接入线上扣款？

当前建议：从履约成功后进入可清算；第一版生成供货应付/服务费账单和清算凭证，不做平台代收打款。

## 5. 店铺归属参数

1. 买家从某个店铺进入后，是否需要长期绑定该店铺？
2. 绑定有效期是否为 30 天？
3. 买家从另一个店铺进入时，是否覆盖原绑定？
4. 买家直接进入平台自营店购买时，是否归属平台自营？

当前建议：订单归属在下单时固化；是否做 30 天绑定由业务最终确认。

## 6. 自有商品参数

已确认：商户可以提交自有商品，必须经过平台审核后才可售卖。

仍需补充：

1. 是否所有审核通过的商户都可以提交自有商品？
2. 是否需要根据保证金余额限制自有商品数量或销售额度？
3. 自有商品是否全部由商户履约？
4. 平台是否需要对自有商品设置最低价？
5. 自有商品是否需要额外类目或资料审核？

## 7. 履约方式参数

1. 哪些商品自动卡密发放，哪些商品手工交付？
2. 是否需要对接第三方发放接口？
3. 权益是否支持撤销？
4. 权益是否有有效期？
5. 发放失败后自动重试几次？

## 8. 退款规则参数

1. 已发放但未使用的权益，是否支持全额退款？
2. 已使用或已激活的权益，是否默认不退？
3. 部分使用后是否支持部分退款？
4. 用户原因退款是否允许？
5. 平台、一级、二级、三级、用户混合责任如何分摊？

## 9. 客服和售后参数

1. 用户默认先联系商户客服还是平台客服？
2. 商户多久不处理售后会自动升级到平台？
3. 平台介入后是否拥有最终裁决权？
4. 商户售后失责是否扣保证金？
5. 客服二维码展示是否需要平台审核？

## 10. 价格风控参数

1. 是否需要最高销售价？
2. 是否需要价格异常提醒？
3. 商户是否可以做临时活动价？
4. 活动价是否同样不能低于最低限价和上游供货价？

## 11. H5 与服务器参数

1. H5 店铺域名使用哪个主体和哪个域名？
2. API 域名、H5 域名、后台域名是否同一备案主体？
3. 服务器优先选择腾讯云、阿里云还是其他国内云？
4. H5 用户登录优先使用手机号短信、邮箱验证码，还是网页登录？
5. 对象存储使用腾讯云 COS、阿里云 OSS 还是其他方案？
6. 生产环境 PostgreSQL、Redis、备份、监控由谁维护？

## 12. 三级以内渠道参数

1. 一级开通下级供货能力需要满足什么条件？
2. 一级开通下级能力是否需要额外保证金？
3. 二级/三级保证金金额是多少？
4. 一级可给二级开放全部商品，还是逐个商品授权？
5. 二级可给三级开放全部商品，还是逐个商品授权？
6. 一级/二级转供价是否必须不低于平台最低销售价，还是只需不低于上游供货价？
7. 二级/三级最终售价是否需要最高价或价格异常提醒？
8. 一级责任、二级责任、三级责任、混合责任的仲裁标准如何细化？

## 13. 收款与支付参数

1. P0 收款通道是否先全部采用商户提交二维码/链接 + 后台确认收款？
2. 真实支付优先接入支付宝、微信商户支付，还是继续商户个人码收款？
3. 商户收款通道是否需要每日限额和单笔限额？
4. 收款凭证是否需要买家上传？
5. mock 支付在生产环境是否强制关闭并由部署检查拦截？

## 14. 平台自营参数

1. 平台自营店是否与渠道店共用同一套店铺装修字段？
2. 平台自营商品履约成本如何录入？
3. 支付通道费是否按真实渠道费率计算，还是先配置为 0？
4. 平台自营经营看板需要哪些指标？
5. 平台自营退款时通道费和履约成本如何归集？
