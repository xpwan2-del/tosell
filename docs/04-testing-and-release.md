# 测试验收与上线清单

## 1. 测试原则

本项目的测试重点不是页面是否能打开，而是交易、履约、退款、结算和数据隔离是否正确。

所有核心测试都必须覆盖金额计算、订单归属、状态变化和操作日志。

## 2. 核心链路测试

### 2.1 代理店铺购买链路

测试步骤：

1. 平台创建商品。
2. 代理选择商品上架。
3. 代理设置销售价。
4. 用户通过代理店铺链接进入。
5. 用户下单支付。
6. 支付成功后订单生成。
7. 平台履约。
8. 用户查看权益。
9. 代理查看订单和收益。

验收标准：

1. 订单归属到正确代理。
2. 用户支付金额等于代理售价。
3. 平台供货价、服务费、代理收益计算正确。
4. 履约成功后订单状态正确。
5. 代理只能看到自己的订单。

### 2.2 定价链路

测试场景：

1. 代理设置价格低于最低限价。
2. 代理设置价格等于最低限价。
3. 代理设置价格高于最低限价。
4. 平台调整最低限价。
5. 平台下架商品。

验收标准：

1. 低于最低限价保存失败。
2. 等于或高于最低限价保存成功。
3. 商品下架后不能购买。
4. 历史订单金额不受价格调整影响。

## 3. 支付测试

测试场景：

1. 支付成功。
2. 用户取消支付。
3. 支付失败。
4. 支付回调重复。
5. 支付回调延迟。
6. 订单超时未支付关闭。

验收标准：

1. 支付成功后订单只处理一次。
2. 取消支付不生成已支付订单。
3. 重复回调不会重复履约。
4. 金额以后端订单金额为准。
5. 支付状态和订单状态一致。

## 4. 履约测试

测试场景：

1. 自动发放成功。
2. 自动发放失败。
3. 人工补发。
4. 发放重试。
5. 重复发放拦截。
6. 退款后权益撤销。

验收标准：

1. 每次发放有凭证。
2. 发放失败有原因。
3. 补发有操作日志。
4. 不允许重复发放同一权益。
5. 用户端能看到权益状态。

## 5. 退款测试

测试场景：

1. 未履约全额退款。
2. 已履约未使用退款。
3. 已使用后申请退款。
4. 平台责任退款。
5. 代理责任退款。
6. 用户原因退款。
7. 部分退款。
8. 退款中取消。
9. 退款失败重试。

验收标准：

1. 退款金额不得超过用户实付金额。
2. 退款中订单不能进入结算。
3. 平台责任和代理责任必须记录。
4. 代理收益扣减正确。
5. 平台供货部分扣减正确。
6. 服务费处理符合规则。

## 6. 结算测试

测试场景：

1. 履约成功后 T+1 进入可结算。
2. 支付成功但未履约不结算。
3. 退款中订单冻结。
4. 风控订单冻结。
5. 投诉订单冻结。
6. 已结算订单不重复结算。
7. 已结算订单后续退款产生追扣。
8. 生成结算单。
9. 导出人工打款文件。

验收标准：

1. 可结算订单筛选准确。
2. 冻结订单不会进入结算单。
3. 结算单金额等于订单明细汇总。
4. 同一订单不能重复结算。
5. 追扣记录可追溯。

## 7. 保证金测试

测试场景：

1. 未缴纳保证金的代理尝试销售。
2. 保证金缴纳后店铺开通。
3. 退款追扣使用保证金。
4. 投诉赔付扣减保证金。
5. 保证金不足限制结算、暂停人工打款或限制销售。
6. 代理退出后退还保证金。

验收标准：

1. 未缴纳保证金不能销售。
2. 扣减必须有原因和记录。
3. 保证金不足时触发限制。
4. 退还保证金前检查未完成订单和售后。

## 8. 权限测试

测试场景：

1. 用户访问他人订单。
2. 代理访问其他代理订单。
3. 代理访问平台财务数据。
4. 运营执行财务结算操作。
5. 财务修改商品信息。
6. 管理员冻结店铺。

验收标准：

1. 用户只能访问自己的订单。
2. 代理只能访问自己的店铺数据。
3. 平台角色权限隔离。
4. 敏感操作必须有日志。

## 9. 财务对账测试

每日对账字段：

1. 用户实付金额。
2. 支付渠道流水。
3. 退款金额。
4. 平台供货价金额。
5. 平台服务费。
6. 代理差价收益。
7. 代理冻结金额。
8. 代理已结算金额。
9. 追扣金额。
10. 保证金余额。

验收标准：

1. 订单明细与汇总金额一致。
2. 支付流水与订单支付金额一致。
3. 退款流水与订单退款金额一致。
4. 结算单与代理收益明细一致。

## 10. 上线前产品清单

1. 商品售卖规则确认。
2. 退款规则确认。
3. 结算规则确认。
4. 保证金规则确认。
5. 代理协议准备。
6. 用户购买协议准备。
7. 隐私协议准备。
8. 虚拟商品使用说明准备。
9. 售后规则准备。
10. 客服话术准备。

## 11. 上线前微信小程序清单

1. 小程序主体确认。
2. 小程序类目确认。
3. 虚拟商品支付能力确认。
4. 微信支付配置确认。
5. 隐私协议配置。
6. 客服入口配置。
7. 分享路径测试。
8. 代理店铺路径测试。
9. 提审材料准备。

## 12. 上线前后台清单

1. 管理员账号创建。
2. 运营账号创建。
3. 财务账号创建。
4. 代理审核流程可用。
5. 商品审核流程可用。
6. 订单查询可用。
7. 售后处理可用。
8. 结算导出可用。
9. 风控冻结可用。
10. 操作日志可查。

## 13. 灰度试运营清单

建议先小范围试运营：

1. 3-5 个测试代理。
2. 5-10 个标准平台商品。
3. 每个代理至少完成 1 单真实购买。
4. 至少测试 1 单真实退款。
5. 至少生成 1 次结算单。
6. 财务完成 1 次人工对账。
7. 客服完成 1 次售后演练。

灰度通过标准：

1. 支付、履约、退款、结算全链路跑通。
2. 财务金额无明显差异。
3. 代理能理解收益和结算规则。
4. 用户能顺利拿到权益。
5. 未出现重大投诉、违规宣传或履约失败。
