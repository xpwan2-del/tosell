# 测试验收与上线清单

本文是 P0 测试依据。当前交付目标为 H5 独立虚拟商品小店系统，不再以微信小程序提审、微信授权登录或小程序支付作为 P0 验收内容。

## 1. 测试原则

1. 测试重点是数据库、API、后台、H5 前台、履约、售后、供货清算、审计和权限隔离是否正确。
2. 所有金额以后端订单快照、退款快照、供货清算明细和账务流水为准。
3. 生产运行代码不得依赖固定 demo id、内存数据、固定收款码或固定店铺切换。
4. 当前支付账号未开通时，P0 验收“商户收款通道绑定 + 确认收款 + 履约状态机”；mock 只能用于开发环境。
5. 禁止出现四级渠道、返佣、返利、团队奖、邀请奖励、代理等级和按招募人数或团队订单计算收益。

## 2. 核心链路测试

### 2.1 平台自营购买链路

步骤：

1. 平台配置自营店和自营收款通道。
2. 平台创建自营商品、详情、图片、库存或卡密。
3. 买家从 H5 自营店进入、下单、选择收款通道。
4. 系统确认收款后自动发码或进入人工交付。
5. 买家查询订单、提取卡密或联系人工客服。
6. 平台后台查看订单、履约、售后、收入、成本和对账。

验收：

1. 订单归属为 `platform_self_operated`。
2. 平台自营订单不生成渠道供货清算。
3. 自营商品、收款通道、客服和卡密全部来自数据库。

### 2.2 一级商户购买链路

验收：

1. 一级商户通过平台邀请码注册并审核入驻。
2. 平台后台可手工创建一级商户和店铺，并生成账号密码。
3. 平台后台手工创建入口只能创建一级商户，不能绕过邀请码创建二级/三级。
4. 一级未确认保证金前不能选品上架、不能销售、不能代理平台商品。
5. 人工确认保证金后必须记录金额、凭证、操作人、确认时间、备注、保证金流水、财务流水和审计日志。
6. 一级确认保证金后只能看到平台开放给自己的商品和平台给自己的供货价。
7. 一级选品上架后，买家在一级店铺看到该商品并按一级售价购买。
8. 订单快照固化平台供货价、一级售价、服务费和一级差价。
9. 一级不能查看无关商户的店铺、订单、收款通道和客户数据。

### 2.3 二级商户购买链路

验收：

1. 二级通过一级邀请码注册，归属到该一级名下。
2. 一级给二级开放商品并设置一级转供价。
3. 二级未确认保证金前不能代理一级商品、不能上架销售、不能配置给三级的转供价。
4. 二级确认保证金后能看到商品资料和一级给自己的转供价。
5. 二级不能看到平台给一级的供货价。
6. 二级上架到自己的店铺后，买家从二级店铺购买。
7. 订单快照可用于清算一级价差和二级销售差价，但二级 API/后台/导出不泄露平台供货价。

### 2.4 三级商户购买链路

验收：

1. 三级通过二级邀请码注册，归属到该二级名下。
2. 二级给三级开放商品并设置二级转供价。
3. 三级未确认保证金前不能代理二级商品和上架销售。
4. 三级确认保证金后能看到商品资料和二级给自己的转供价。
5. 三级不能看到一级转供价和平台供货价。
6. 三级不能创建四级邀请码或四级商户。

## 3. 逐级价格隔离专项

必须测试：

1. 平台账号可查看平台供货价、一级转供价、二级转供价和最终售价。
2. 一级账号可查看平台给自己的供货价、自己给二级的转供价和自己店铺售价。
3. 二级账号只可查看一级给自己的转供价、自己给三级的转供价和自己店铺售价。
4. 三级账号只可查看二级给自己的转供价和自己店铺售价。
5. H5 买家只可查看最终销售价。
6. API 响应、后台表格、订单列表、导出文件和浏览器网络响应均不得泄露越级价格。

失败标准：

1. 二级看到 `platform_supply_price_cents`。
2. 三级看到 `platform_supply_price_cents` 或 `first_tier_supply_price_cents`。
3. 前端隐藏字段但接口仍返回敏感价格。

## 4. 收款通道测试

测试场景：

1. 商户提交支付宝个人收款码，平台审核通过，H5 支付页展示。
2. 商户提交支付宝商户码/链接，平台审核通过，H5 支付页展示二维码或跳转按钮。
3. 商户提交微信个人收款码，平台审核通过，H5 支付页展示。
4. 商户提交微信商户码/链接，平台审核通过，H5 支付页展示二维码或跳转按钮。
5. 禁用或拒绝的通道不在 H5 展示。
6. 后台或后续回调确认收款后，订单进入已支付并触发履约。

验收：

1. 收款通道绑定到当前店铺和商户。
2. 订单保存收款通道快照。
3. 生产环境不展示固定 mock 通道。

## 5. 卡密与履约测试

1. 自动卡密商品：确认收款后扣减库存、生成权益；是否需要提取码由商品后台配置决定。
2. 手工交付商品：支付成功后显示客服入口和待交付状态，不显示卡密。
3. 买家邮箱为选填；未填写邮箱也能下单，填写邮箱则确认收款发码后同步发送激活码/卡密邮件。
4. 启用提取码的商品，下单必须填写提取码，提取码哈希存储。
5. 未启用提取码的商品，下单页不强制提取码，订单详情通过身份/订单凭证展示权益。
6. 提取页只输入提取码即可查看，订单上下文来自确认成功跳转或安全订单查询凭证。
7. 提取码错误 3 次锁定 30 分钟，锁定期内继续请求被拒绝。
8. 买家允许重复查看已发放卡密。
9. 退款完成后禁止继续查看卡密。
10. 每次提取成功、失败、锁定、退款后拒绝都写入提取日志。
11. 重复确认收款或重复回调不得重复发码。

## 6. 优惠券测试

1. 新用户首次注册登录获得无门槛赠送券。
2. 平台后台可创建券模板、限制适用商品、配置有效期、发放、作废和查看使用记录。
3. 优惠券可限制适用商品和有效期。
4. 优惠券不可叠加。
5. 过期、作废、已使用优惠券不可再用。
6. 商品范围不匹配时后端拒绝用券。
7. 下单金额以后端优惠计算为准。
8. 平台券抵扣不影响供货价、转供价和各级价差结算基准。
9. 退款完成后已使用券作废，不退回买家券包。

## 7. 后台与前台联动测试

1. 后台创建商品，H5 店铺列表和详情出现。
2. 后台下架商品，H5 不可购买。
3. 后台修改店铺名、公告、客服二维码，H5 同步。
4. 后台导入卡密，H5 显示可售库存。
5. 后台修改收款通道，H5 支付页同步。
6. 商户后台越权访问其他商户数据返回 403/404 或空数据。
7. 后台优惠券配置后，H5 下单页只展示当前买家可用券。
8. 平台销售统计能按自营、一级、二级、三级、商品和店铺维度展示 GMV、用户实付、平台券补贴、退款、服务费、各级价差、待清算和已清算。

## 8. 上线前清单

1. `npm test` 通过。
2. `npm run typecheck` 通过。
3. `npm run db:validate` 通过。
4. 平台后台、商户后台和 H5 前台使用同一数据库 API。
5. PostgreSQL、对象存储、HTTPS、日志、备份、监控方案确认。
6. mock 支付仅开发环境可用，生产环境部署检查拦截。
7. 硬编码扫描通过：生产运行代码不得依赖 `shop-1`、`agent-1`、`prod-1`、固定收款码或固定商品内容。
8. 权限越权测试和价格泄露测试通过。
9. 后台手工创建二级/三级商户被后端拒绝并写审计。
10. 人工确认保证金生成保证金流水、财务流水和审计日志。

## 9. 回滚或暂停标准

1. 出现金额计算错误、重复入账、重复履约、清算重复或退款超额。
2. 出现用户、商户、店铺、订单、收款通道、卡密或价格越权泄露。
3. 二级能看到平台给一级的供货价。
4. 三级能看到一级转供价或平台供货价。
5. 三级能创建四级邀请码或四级商户。
6. 平台自营订单错误生成渠道供货清算。
7. 生产环境出现固定 mock 支付或固定测试店铺。
