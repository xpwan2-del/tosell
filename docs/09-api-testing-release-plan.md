# API、测试与上线计划

本文是 P0 开发依据，定义 H5 虚拟商品小店的 API 分组、接口契约、无硬编码红线、错误码、幂等、测试矩阵、灰度和上线前验证。

## 1. API 契约总原则

1. 后端是金额、状态、库存、卡密、收款、退款、供货清算、保证金和账务的唯一权威。
2. 前端不得提交可信金额。售价、实付、服务费、收益、退款、清算都必须由后端计算或校验。
3. 金额统一使用 `amount_cents`，服务费率默认 `service_fee_bps=50`。
4. 商户端 API 必须从认证主体派生 `merchant_id/shop_id`，不能信任前端传参。
5. H5 买家 API 必须从登录态、订单查询凭证或店铺上下文派生身份和店铺，不得信任前端传入 `user_id`。
6. 导出 API 必须校验后台角色权限并写审计。
7. 不得提供邀请奖励、团队业绩、代理等级、返佣比例、团队订单奖励 API。
8. 渠道供货 API 最多允许平台 -> 一级 -> 二级 -> 三级，不得提供四级能力。
9. 商品资料可向下级开放；越级供货价格必须由服务端响应整形层过滤。
10. 生产 API 不得依赖内存 demo store、固定 shop id、固定 agent id、固定商品、固定收款码或固定 mock 结果。

## 2. API 分组

### 2.1 H5 买家 API

1. 店铺详情：`GET /api/h5/shops/:slug`，返回店铺名称、Logo、公告、客服入口、营业状态。
2. 商品列表：`GET /api/h5/shops/:slug/products`，返回当前店铺可售商品。
3. 商品详情：`GET /api/h5/products/:id`，返回售价、图片、详情、库存、履约方式、售后规则。
4. 注册/登录：手机号、邮箱或验证码登录，首次注册触发赠券。
5. 创建订单：后端校验店铺、销售商户保证金确认状态、商品、库存、卡密、优惠券、收款配置、价格和风控；保证金未确认时拒绝创建可支付订单。
6. 收款配置列表：只返回当前店铺已启用通道。
7. 订单详情：买家只能访问自己的订单或通过安全订单查询凭证访问。
8. 买家优惠券：只返回当前买家可用券、不可用原因、适用商品和有效期。
9. 卡密提取：仅对后台配置启用购买密码的商品开放；独立提取页只输入购买密码，订单上下文来自安全跳转 token、订单查询凭证或登录态订单详情；错误 3 次锁定 30 分钟，退款后不可查看。
10. 邮箱发码：邮箱为选填；填写邮箱的订单在确认收款并发放权益后发送激活码/卡密邮件。邮箱发码不是唯一取码方式，未填写邮箱不得影响下单和提取。
11. 售后申请和售后详情。

H5 买家 API 禁止返回：供货价、转供价、差价、服务费、供货清算、风控、其他店铺切换入口。

### 2.2 商户 API

1. 入驻申请、审核状态、保证金账户和流水。
2. 店铺详情、店铺装修、客服二维码、分享链接。
3. 收款配置提交、审核状态、启用、禁用、默认通道。
4. 可选商品列表：保证金未确认时返回不可选品状态和原因。
5. 商品上架、下架、改价、自有商品提交：保证金未确认时拒绝上架和代理商品。
6. 库存和卡密批量导入、库存列表、手工交付。商户自有自动卡密商品必须走商户 scoped 库存 API；平台 admin 卡密接口只用于平台商品或平台代操作并留审计。
7. 订单列表、订单详情、确认收款、履约处理、售后处理。
8. 邀请码创建和列表：三级不能创建。
9. 一级给二级开放商品和设置一级转供价。
10. 二级给三级开放商品和设置二级转供价。
11. 收益、冻结金额、供货应付/服务费账单、追扣记录。
12. 销售统计：仅返回当前商户权限范围内的 GMV、用户实付、平台券补贴、退款、服务费、价差、待清算和已清算。

商户 API 必须按角色脱敏：

1. 一级可看到平台给自己的供货价。
2. 二级只看到一级给自己的转供价，不返回平台供货价。
3. 三级只看到二级给自己的转供价，不返回一级转供价和平台供货价。

### 2.3 平台后台 API

1. 商户审核、保证金管理、店铺管理。
2. 后台手工创建一级商户和店铺，生成初始账号密码并记录交付状态；接口只能创建一级商户，传入二级/三级目标必须拒绝并记录审计。
3. 平台商品库、商品图片、详情、价格、上下架。
4. 自有商品审核。
5. 收款配置审核和风控。
6. 邀请码、渠道关系、转供价审计。
7. 订单、收款确认、履约、卡密、售后、退款。
8. 供货应付、服务费账单、清算凭证、追扣、保证金扣减。
9. 平台自营店、自营商品、自营订单和自营对账。
10. 优惠券模板、适用商品、发放、使用、作废、退款后券作废。
11. 买家用户、订单查询凭证、提取日志、锁定状态和风控查看。
12. 销售统计：自营、一级、二级、三级、商品和店铺维度的 GMV、用户实付、平台券补贴、退款、服务费、各级价差、待清算和已清算。
13. 审计日志、导出、基础看板。

平台后台可以按 RBAC 查看完整价格链，但所有查看、导出和敏感操作必须记录审计。

### 2.4 收款与回调 API

P0 当前以商户收款配置绑定为主：

1. 创建支付记录：绑定订单和店铺收款配置快照。
2. 商户或平台确认收款：幂等更新订单为已支付。
3. 后续真实支付回调：预留验签、金额校验、原始 payload、幂等处理。

要求：

1. 金额必须等于订单应付金额。
2. 重复确认或重复回调不得重复履约、重复发码、重复入账。
3. 生产环境不能默认展示 mock 支付。

### 2.5 导出 API

1. 订单明细导出。
2. 收款记录导出。
3. 退款对账导出。
4. 供货应付和服务费账单导出。
5. 清算凭证导出。
6. 商户收益明细导出。
7. 保证金流水导出。
8. ledger 明细导出。
9. 审计日志导出。

导出必须按角色做字段脱敏。二级导出不得出现平台供货价；三级导出不得出现平台供货价和一级转供价。

## 3. 无硬编码 API 红线

生产运行代码禁止：

1. API 使用固定 `shop-1`、`agent-1`、`prod-1`、`channel-rel-1`、`ap-code` 作为业务来源。
2. H5 前台硬编码商品、详情、图片、价格、库存、客服二维码、收款二维码。
3. 后台按钮固定操作某个测试商品、测试店铺、测试商户或测试订单。
4. 后端以内存 demo store 作为生产数据来源。
5. mock 支付在生产环境可见或可调用。
6. 前端本地常量决定平台供货价、转供价、优惠券、服务费、清算金额。

允许：

1. 单元测试、集成测试、E2E 测试中的 fixture。
2. Prisma seed 和开发种子商品。
3. `.env.example` 示例值。
4. 文档中的业务示例。

硬编码扫描必须进入上线门槛。

## 4. 错误码原则

错误码至少区分：

1. 参数错误。
2. 认证失败。
3. 权限不足。
4. 资源不存在。
5. 状态不允许。
6. 金额校验失败。
7. 低于最低限价。
8. 低于上游供货价。
9. 重复请求。
10. 风控限制。
11. 保证金不足。
12. 收款配置不可用。
13. 库存或卡密不足。
14. 购买密码错误或锁定。
15. 系统内部错误。

错误信息可支持前端展示，但不得泄露其他商户、后台财务、风控策略或越级供货价。

## 5. 幂等与唯一约束

1. 确认收款：`payment_confirm:{payment_id}:{confirm_source}`。
2. 真实支付回调：`pay:{channel}:{channel_trade_no}`。
3. 退款回调：`refund:{channel}:{channel_refund_no}`。
4. 履约尝试：`fulfill:{order_item_id}:{attempt_no}`。
5. 权益发放：`entitlement:{order_item_id}:{virtual_code_id}`。
6. 卡密提取：不重复发放，只读取已绑定权益。
7. 供货清算生成：`clearing:{merchant_id}:{period_start}:{period_end}:{batch_no}`。
8. 清算明细：`clearing_items(order_id, clearing_role)`。
9. 追扣：`clawback:{source_type}:{source_id}:{merchant_id}`。
10. 保证金交易：`deposit:{type}:{source_type}:{source_id}:{merchant_id}`。
11. 审计事件：`audit:{request_id}`。

## 6. 验收测试矩阵

| 模块 | 关键场景 | 预期结果 |
| --- | --- | --- |
| 无硬编码 | 扫描生产运行代码中的固定店铺、商品、商户、收款码、mock 支付 | 生产路径无硬编码业务数据 |
| 店铺联动 | 后台改店铺名、公告、客服二维码 | H5 当前店铺同步变化 |
| 商品联动 | 后台新增/改价/下架商品 | H5 列表和详情同步，历史订单快照不变 |
| 收款配置 | 商户提交二维码/链接，平台审核启用 | H5 只展示当前店铺有效通道 |
| 自动卡密 | 确认收款后自动发码 | 扣库存、生成权益；启用购买密码时可提取，未启用时订单详情查看 |
| 手工交付 | 支付成功后商户手工处理 | 买家看到客服入口，不看到卡密 |
| 购买密码 | 商品后台配置启用、独立提取页只输入购买密码、错误 3 次、30 分钟锁定、退款后查看 | 服务端用 token/订单查询凭证派生订单上下文，记录日志，退款后不可查看 |
| 邮箱发码 | 买家选填邮箱 | 填写邮箱才发送激活码/卡密，未填写不影响下单和独立提取 |
| 注册赠券 | 新用户首次注册 | 自动发放无门槛赠券 |
| 优惠券 | 商品限制、有效期、作废、不可叠加、退款后作废 | 后端正确校验，平台券不影响价差 |
| 销售统计 | 自营/一级/二级/三级/商品/店铺维度 | GMV、用户实付、平台券补贴、退款、服务费、价差、待清算、已清算正确且脱敏 |
| 邀请码 | 平台/一级/二级邀请码注册 | 层级归属正确，三级不能建四级 |
| 后台手工开店 | 平台手工创建一级商户和店铺 | 生成账号密码，状态待保证金确认，写审计 |
| 后台手工开店越权 | 平台手工创建接口传入二级/三级目标 | 后端拒绝，不能绕过邀请码关系，并写审计 |
| 保证金门槛 | 未确认保证金时选品、上架、代理商品、创建订单 | 后端拒绝并返回明确原因 |
| 人工确认保证金 | 平台后台线下确认保证金 | 记录金额、凭证、操作人、确认时间、备注，生成保证金流水、财务流水/ledger 和审计日志 |
| 一级选品 | 一级选择平台商品 | 一级可见平台给自己的供货价 |
| 二级选品 | 一级开放商品给二级 | 二级可见商品和一级转供价，不见平台供货价 |
| 三级选品 | 二级开放商品给三级 | 三级可见商品和二级转供价，不见一级转供价/平台供货价 |
| 价格泄露 | API、后台表格、导出、浏览器网络响应 | 按角色脱敏 |
| 下单 | 篡改价格、商品、店铺、优惠券 | 后端重新计算或拒绝 |
| 确认收款 | 重复确认、金额不一致 | 幂等；金额不一致拒绝 |
| 供货清算 | 商户自收款、退款中、风控、重复生成 | 只生成合规供货应付/服务费账单，不做平台 T+1 代打款，不重复 |
| 权限隔离 | 商户跨店查询、买家查他人订单 | 403/404 或空数据 |
| 平台自营 | 自营店下单、履约、退款 | 不生成渠道供货清算 |

## 7. 安全与越权专项

P0 必测：

1. 篡改 `shop_id/merchant_id` 创建订单。
2. 篡改 `sale_price_cents/paid_amount_cents/service_fee_cents`。
3. 商户 A 查询商户 B 的订单、客户、商品、收款配置、供货清算、售后。
4. 买家 A 查询买家 B 的订单和卡密。
5. 二级查询接口返回平台给一级的供货价。
6. 三级查询接口返回一级转供价或平台供货价。
7. 三级尝试创建四级邀请码。
8. 生产环境访问 mock 支付或固定测试收款码。
9. 前端隐藏敏感字段但接口仍返回敏感价格。

任一失败都不能上线。

## 8. 上线前全链路验证

上线前必须完成：

1. 平台创建商品、详情、图片、供货价、最低价、卡密池。
2. 平台创建或审核至少 1 个一级、1 个二级、1 个三级商户。
3. 一级选品上架，二级选品上架，三级选品上架。
4. 每个商户绑定并审核至少 1 个收款配置。
5. H5 当前店铺完成下单、确认收款、自动发码、跳转独立提取页并只输入购买密码提取卡密。
6. 手工交付商品完成下单、确认收款、客服交付记录。
7. 注册赠券和商品优惠券完整验证，包括商品范围、过期、作废、不可叠加、平台券不影响价差和退款后券作废。
8. 售后退款、退款后禁止查看卡密、供货清算冻结完整验证。
9. 供货应付、服务费账单、清算凭证、已清算后追扣完整验证；不做平台代收后 T+1 给商户打款。
10. 平台自营店完整下单、履约、退款和对账。
11. 逐级价格隔离专项通过。
12. 硬编码扫描通过。

## 9. 发布阶段

1. 阶段 0：文档、原型和口径定稿。
2. 阶段 1：数据库与后端主链路。
3. 阶段 2：平台后台和商户后台。
4. 阶段 3：H5 买家店铺前台。
5. 阶段 4：卡密、优惠券、售后、供货清算、风控。
6. 阶段 5：联调、越权测试、价格泄露测试、硬编码扫描、视觉验收。
7. 阶段 6：生产部署准备，补真实支付/回调。

## 10. 回滚或暂停标准

1. 出现金额计算错误、重复入账、重复履约、清算重复、退款超额。
2. 出现商户或买家越权访问。
3. 二级看到平台供货价。
4. 三级看到一级转供价或平台供货价。
5. 生产代码依赖固定测试数据或固定收款码。
6. mock 支付在生产环境可见。
7. 三级能继续创建四级渠道。
8. 平台自营订单错误生成渠道供货清算。
