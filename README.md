# 微商小店虚拟商品供货平台

本项目当前目标已调整为 H5 独立虚拟商品小店系统，包含平台后台、商户后台和多商户独立店铺前端。项目不再以微信小程序为交付目标，不再依赖微信授权登录；微信和支付宝只作为可选收款/支付方式。

平台既可以自己开设平台自营店直接销售，也可以向一级、二级、三级商户供货；商户审核入驻并缴纳保证金后，可以拥有自己的小店、客服入口、商品售价、收款配置和推广链接。用户可以在平台自营店或商户小店购买虚拟商品。熟人或内部商户允许平台后台手工创建一级商户和店铺，并生成账号密码交付；不熟悉的商户必须通过入驻申请页输入邀请码注册并等待审核。

本项目不做返佣分销、团队业绩奖、邀请奖励或拉人头收益。平台自营与受控三级以内商户价差供货并存：平台自营店 -> 消费者；平台供货 -> 一级商户 -> 二级商户 -> 三级商户 -> 消费者。商户收益来自供货价差和转供价差，不按招募人数、团队订单或返佣比例计算；三级商户不得继续发展四级。

## 硬编码红线

生产运行代码不得硬编码店铺、商户、商品、价格、库存、卡密、客服二维码、收款码、支付链接、商户供货关系或 mock 支付结果。后台、商户后台和 H5 前台必须共用数据库/API；后台上架、改价、导入卡密、配置客服和收款配置后，前台必须从接口同步展示。收款配置底表统一为 `collection_payment_configs`，不再使用旧 `shop_collection_channels`。

允许硬编码的范围仅限测试 fixture、Prisma seed、`.env.example` 示例和文档示例。

## 第一版核心闭环

平台自营闭环：平台配置自营店 -> 上架自营商品 -> 用户下单 -> 选择当前可用支付方式 -> 系统确认收款或支付成功 -> 虚拟权益发放 -> 退款/售后处理 -> 平台收入/成本对账。

商户供货闭环：商户入驻审核 -> 缴纳保证金 -> 配置独立小店 -> 选择平台商品或提交自有商品审核 -> 设置销售价/转供价 -> 分享店铺 -> 用户下单 -> 选择当前店铺可用支付方式 -> 系统确认收款或支付成功 -> 虚拟权益发放 -> 退款/售后处理 -> 供货应付/服务费/追扣对账。

保证金是销售和代理商品的硬门槛。未确认缴纳保证金的商户不能销售、不能选择平台商品上架、不能代理上级商户开放的商品、不能配置转供价。平台后台可以人工确认保证金缴纳状态，并必须记录操作人、金额、凭证和审计日志。

## 第一版不做

- 四级商户或更多层级
- 下级返佣
- 团队奖、平级奖、领导奖
- 邀请奖励、拉人奖励
- 复杂 CRM
- 复杂营销活动
- 自动化私域运营
- 开放平台 API
- 平台代收后给商户做 T+1 打款结算

## 当前阶段

当前已进入 H5 P0 生产化实现阶段。仓库包含：

- `packages/database`：Prisma 数据模型、种子数据入口。
- `packages/core`：金额计算、退款拆账、供货结算筛选、权限隔离、幂等工具和单元测试。
- `apps/api`：Fastify API 服务，必须改为数据库驱动的店铺/商品、订单、收款、履约、退款、结算、保证金、风控、审计和权限服务。
- `apps/admin`：平台后台/运营财务工作台骨架。

历史 V2 文档仅作参考。当前优先完成 H5、数据库、后台、商户后台、收款配置、卡密、逐级价格隔离、供货应付/服务费对账和硬编码清理。e支付按已提供商户资料配置自动确认；支付宝/微信商户支付先完成后台配置、签名、回调和查单接口准备，待真实商户号、密钥、证书和回调域名具备后即可联调启用。

## 本地命令

```bash
npm install
npm test
npm run typecheck
npm run db:validate
npm run api:dev
npm run admin:dev
```

`db:validate` 默认使用示例 `DATABASE_URL` 校验 Prisma schema。真实环境需要按 `.env.example` 配置 PostgreSQL、H5 域名、后台账号、对象存储和支付/收款配置参数。

## API 本地联调

```bash
npm run api:dev
```

当前本地仍可能存在历史内存 demo store；这只允许作为开发过渡，不能作为 P0 生产实现。正式环境必须使用 Prisma/PostgreSQL 事务服务、真实登录态、数据库收款配置和服务端权限隔离。mock 认证头：

- 用户端：`x-user-id`
- 商户端：`x-merchant-id`、`x-shop-id`
- 后台：`x-admin-id`、`x-admin-role: operator | finance | admin`

主要 MVP 分组：

- `/api/user/*`：店铺、商品、订单报价/创建、订单详情、售后申请。
- `/api/merchant/*`：入驻、店铺、商品定价、订单收益、供货应付、服务费账单、追扣记录。
- `/api/admin/*`：商户审核、商品、订单、履约、退款拆账、供货账单、服务费账单、保证金、风控、审计。
- `/api/callbacks/*`：历史 mock 回调只允许开发环境使用；P0 生产路径必须使用商户收款配置确认、e支付回调/查单，以及支付宝/微信商户支付的真实回调。
- `/api/exports/*`：基础对账摘要。
